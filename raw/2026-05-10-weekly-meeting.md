#### 정답자 → 제출한 답안 목록 노출하는것

- Figma: <https://www.figma.com/design/z6pBPJzrSSrbfQpnxUUXRi/%F0%9F%A4%94V2.-CS-%ED%80%B4%EC%A6%88?node-id=9642-317&t=bpPN5udo45D5qFlV-11>
- 1. 객관식 — 선택 분포 버블 UI
  - 사람들이 선택한 답안을 버블 크기로 시각화해, 어떤 선택지에 얼마나 몰렸는지 한눈에 보여준다. 정답은 컬러 강조, 오답은 주변에 배치되며 실시간으로 버블 크기가 변화한다. 버블 클릭 시 해당 답을 선택한 사람들의 반응·코멘트·재밌는 답변을 함께 확인할 수 있다.
  - 노출하는 정보
  - 피드백
- 2. 주관식 — 워드 클라우드 그룹 UI
  - 비슷한 답변을 자동으로 묶어 워드 클라우드 형태로 표현한다. 많이 입력된 답변일수록 크게 표시되며, 같은 의미의 다양한 표현도 함께 보여준다. 웃긴 오답이나 독특한 표현은 별도로 강조되어 사람들의 반응을 유도한다.
  - 피드백
- 3. 빈칸넣기(단답형) — 정답 근접도 UI
  - 입력한 답변을 정답과의 유사도 기준으로 중심에서 가까운 위치에 배치한다. 정답과 거의 일치한 답은 중앙에, 오답일수록 바깥쪽에 퍼지도록 시각화한다. 사람들이 어떤 부분에서 헷갈렸는지 직관적으로 확인할 수 있다. (취소선 처리됨)
  - 피드백
- 4. 순서 맞추기 — 흐름 경로(Path) UI
  - 사람들이 입력한 순서를 선(Path) 형태로 시각화해 어떤 흐름으로 답했는지 보여준다. 많이 선택된 순서는 굵고 선명하게, 적은 답변은 얇고 흐리게 표현된다. 특정 단계에서 사람들이 많이 헷갈린 구간도 함께 강조해 보여줄 수 있다.
  - 피드백

### 6. AI의 요약

- 메인페이지는 `MAIT 사용법`과 `주요 기능` 두 섹션으로 앵커를 고정하기로 했다. 각각 `#guide`, `#features`를 쓰고 FE가 두 태그를 반영하는 방향이다.
- 문제셋 생성 흐름은 문제 정보 입력 화면과 생성 시 정보 입력 화면을 통일하는 쪽으로 정리했다. 학습모드/실시간모드 선택은 앞단에 배치하고, 자료 업로드·보충 설명처럼 보여야 하지만 수정하면 안 되는 필드는 읽기 전용으로 노출해야 한다.
- 학습모드/복습모드 풀이 화면은 별도 나가기 버튼이나 브레드크럼을 두지 않는 방향으로 의견이 모였다. 퀴즐렛·퀴즈앤 사례처럼 문제풀이 뷰에서 NavBar만 노출하거나 뒤로가는 경로가 없는 흐름이 일반적이라는 근거가 제시됐다.
- 풀이 중 반응과 답안 노출은 이모지와 유형별 제출 현황으로 분리해 봤다. 이모지는 기본 5개를 두는 쪽으로 정리됐고, 제출 답안은 객관식 버블 UI, 주관식 워드 클라우드, 순서 맞추기 Path UI를 검토하기로 했다.
- 새 기능 안내는 업데이트 후 최초 진입 시 현재 온보딩과 같은 형식으로 간단히 안내하고, 상세 설명이 필요하면 기능 설명 페이지 링크를 붙이는 방향이다. 릴리즈 노트는 별도로 쓰되, 인앱 안내는 다음 기능부터 적용을 고려한다.
- 카테고리 추가 이후 문제셋 목록이나 문제 카드에 카테고리를 노출할지는 디자인 제약을 보고 결정하기로 했다. 현재 디자인에서는 바로 추가가 어려워 보이지만, UI 개선이 들어가면 기존/신규 문제셋 노출 영역에서 함께 검토할 필요가 있다.
- 파트별 할일은 기획서·화면·API·프론트 작업으로 나뉘었다. 기획은 수정된 기획서와 일정 관리, BE는 카테고리·학습모드 대시보드·문제 제목 기본값·팀삭제 데이터, FE는 우승자 5초 종료·복습모드 동선·최근 방문 탭·LNB 펼치기 등을 챙긴다. Wiki Repo 미해결 묶음은 이번 팔로우업 티켓 대상에서 제외했다.
