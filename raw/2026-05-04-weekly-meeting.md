### 6. AI의 요약

- 이번 회의는 5월 12일 배포와 5월 15일 코테이토 사용을 앞두고, 배포 가능한 범위와 남은 개선 항목을 재정렬하는 성격이 강했다. 학습모드, 카테고리, 복습모드 동선, 개인 워크스페이스, 알림처럼 이미 진행 중인 큰 흐름을 유지하되, 실제 사용 전에 막힐 수 있는 진입/결과 확인/알림 이동을 다시 점검했다.
- 등수정보 노출은 기존 기획을 더 구체화했다. 정답자/오답자에게 보여줄 제출 현황, 1등과의 시간 차이, 실시간 참여 인원, 제출 답안 목록, 이모지 피드백이 논의됐고, 우선은 1등과 몇 초 차이인지와 실시간 참여 인원 표기를 먼저 작업 대상으로 잡았다.
- 알림은 수신자와 클릭 후 이동 위치를 역할별로 나눠 정리했다. owner에게는 팀 초대 완료와 팀 탈퇴 알림을 멤버관리로 연결하고, maker/player에게는 팀 삭제나 본인 탈퇴처럼 더 이상 이동할 대상이 없는 경우 이동 없이 안내하며, 인정답안 추가로 오답이 정답이 된 player는 해당 문제의 채점된 화면으로 보내는 방향이다.
- 학습모드는 배포 전 마무리 항목과 배포 이후 개선 항목을 분리했다. 배포건에서는 빠져나가는 동선, 완료 결과 재확인, 채점 이후 디자인 같은 사용 흐름이 빈틈으로 보였고, 별도 개선으로는 관리자뷰에서 팀원 풀이 현황과 결과 통계를 더 잘 보여줄 필요가 제기됐다.
- 카테고리는 구현/전달이 이어지고 있지만, 디자인 쪽에서는 40자 초과 입력을 막기보다 허용하면서 초과 상태를 보여주는 방향이 남았다. LNB의 팀관리 진입은 멤버관리로 기본 이동하는 것으로 정리됐다.
- 개인 워크스페이스와 신규 기능 안내는 사용자가 어디로 돌아오고 무엇을 새로 써야 하는지의 문제로 이어졌다. 최근 방문 탭/팀, 메인 페이지 앵커링, 새 기능 출시 안내 방식이 함께 언급됐고, 일부는 명세 완료/확인 처리됐지만 사용자 안내 방식은 별도 정리가 필요하다.
- 운영 이슈로는 토큰 재발급 실패가 별도 문제로 공유됐고, 학습모드 자동종료 엣지 케이스 문의는 논의한 안대로 진행하기로 했다. 개발 파트에서는 삭제한 팀 하위 데이터 접근 차단, 학습모드 대시보드 관련 API, 카테고리 API, 개인 워크스페이스 공간 생성이 남은 작업으로 정리됐다.
