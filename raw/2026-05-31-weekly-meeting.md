- Discord link: https://discord.com/channels/1260463506642440252/1450872674359316510/1509935965873901769
- 개인 워크스페이스에서는 시작/종료 기능이 제외되므로 아래 기능도 제외하는지 문의했다.
  - 문제 풀이 완료에 따른 자동 종료 처리
  - Maker(본인)의 수동 종료 처리
- 문제 풀이 완료에 따른 자동 종료 처리는 살린다.
- 자동 종료는 가능해야 한다.
- 개인 워크스페이스에서 풀이 완료 후 통계로 전환한다.
- 상세 대시보드에서는 오답률을 노출하지 않는다.
- Maker의 수동 종료는 불가능해야 한다는 맥락으로 정리됐다.

#### [대시보드2차] 문제 셋 노출 정보 문의

- 학습모드에서는 우승자 영역이 존재하지 않는 경우가 있다.
- 문제셋 목록이 모든 문제셋인지, 유저가 푼 문제셋만인지 문의했다.
- 풀이 완료된 문제셋 범위에서, 내가 풀지 않은 문제셋이라도 모든 문제셋을 노출하는 것으로 정리했다.
- 유저가 풀지 않은 문제셋의 노출 정보:
  - `내 정답률: - / 전체 평균 정답률` 형태로 노출한다.
  - 우리팀 랭킹과 오답률 Top3는 그대로 노출한다.
  - 점수는 0점으로 표시한다.
  - 전부 틀린 문제로 반영한다.

#### 정답 데이터 기준이 첫번째 제출 기준인 경우 발생하는 모호함 문의

- 기존 이해:
  - 실시간 풀이 중에는 하나라도 맞으면 player에게 정답/득점으로 인지된다.
  - 통계용으로는 첫번째 제출이 맞아야만 정답으로 인정되는 것으로 이해했다.
  - 이에 따라 대시보드의 정답수 기준 랭킹은 첫번째 제출인 경우의 정답만 노출된다고 이해했다.
- 우려:
  - 실시간 풀이 중에는 정답/득점이었지만 대시보드에서는 사용자가 인지한 것과 카운트가 다를 수 있다.
  - 예: 첫 제출 오답, 두 번째 제출 정답, 득점이면 사용자는 득점자지만 기존 기준상 정답자가 아닐 수 있다.
- 정리:
  - 통계에서는 n번째 제출이라도 득점자라면 카운트한다.
  - 통계 정답 기준은 득점자인지 여부를 먼저 본다. 득점자라면 정답 처리한다.
  - 득점자가 아닌 경우에는 최초 제출 기록 기준으로 오답/정답 처리한다.

- 사용자가 풀지 않은 문제셋의 대시보드 정보는 내 정답률을 `-`, 전체 평균 정답률은 그대로 노출하고, 우리팀 랭킹/오답률 Top3도 유지한다. 상세 값은 점수 0점, 전부 틀린 문제로 반영하는 방향으로 기록됐다.
- 통계용 정답 기준은 실시간 풀이 인지와 대시보드 통계가 어긋나는 문제를 줄이기 위해, 득점자라면 정답으로 처리하고 득점자가 아닌 경우에는 최초 제출 기록 기준으로 정답/오답을 처리하는 방향으로 정리했다.
- 개인 워크스페이스는 시작/종료 기능을 제외하더라도 문제 풀이 완료에 따른 자동 종료 처리는 유지한다. Maker 수동 종료는 불가능해야 하며, 개인 워크스페이스에서 풀이 완료 시 통계로 전환하고 상세 대시보드에서는 오답률을 노출하지 않는다.
- 카테고리/세부 과목처럼 보이던 두 영역은 실제 데이터 기준 `카테고리 이름` 하나로 합치고, 클릭 시 별도 이동은 하지 않는 것으로 확인했다. 카테고리 관리의 체크/취소 또는 취소/저장 상태 표현은 디자인/프론트가 계속 맞춰야 한다.
- 실시간 풀이 등수 노출은 참여자 수가 계속 표시되는 전제에서 문제 영역과 겹치는 문제가 논의됐다. 대안으로 플로팅 버튼을 두고 참여 인원 수, 참여자 닉네임, 이모지를 함께 노출하는 방향이 나왔으며, 실제 데이터 기반 화면 구성과 문제 유형별 기준 명세가 후속으로 남았다.
- 온보딩은 앞/뒤 이동 UI, 다시 보지 않기, 닫기, 순서 기능과 추가 플로우 팝업 처리가 필요하다. 문제입력 통일 UI, 등수 노출 디자인, 문제카드 UI 개선 고민도 각 파트 후속 작업으로 유지됐다.
