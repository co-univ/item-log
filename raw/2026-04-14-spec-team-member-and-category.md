# 팀관리 - 멤버관리, 카테고리 관리

## 1. 멤버 관리

| **구분** | **세부기능** | **상세설명** | **코멘트** |
| --- | --- | --- | --- |
| 권한 관리  | 1.1 사용자 리스트 표시 | 두 개의 컬럼으로 구분: 좌측 `maker`, 우측 `player` 각 영역에는 해당 권한을 가진 사용자 리스트를 표시 체크박스 개별 사용자 선택 용도로 삭제 및 권한 이동 시 사용  승인 대기 멤버 비활성화 상태 오른쪽에 승인/거절 버튼 노출 리스트 최상단에 위치 승인 완료 멤버 활성화 상태 오른쪽에 권한 텍스트를 표시 (owner/maker/player) 하단에 위치 owner는 maker 영역 하단에 별도 표시 |  |
|  | 1.2 권한 승인/거절 | maker 초대 시 승인 절차 존재 가능 `승인` 버튼 클릭 시 확인 모달 표시 "000님의 maker 권한을 승인/거절하시겠습니까?" 확인 시 상태가 `승인됨`으로 변경 거절 시 목록에서 제거 |  |
|  | 1.3 권한 전환 | 중앙의 `>>` (player로 전환), `<<` (maker로 전환) 버튼 클릭 시 역할 변경 사용자 선택(체크박스) 후 버튼 활성화 변경 후 각 리스트 자동 갱신 |  |
|  | 1.4 사용자 삭제 | maker/player 리스트에서 여러 사용자를 체크박스로 선택 후 삭제 버튼 클릭 시 동작 사용자 선택(체크박스) 후 버튼 활성화 삭제 클릭 시 공통 삭제 모달 표시 문구: "선택한 계정을 삭제하시겠습니까?" 확인 시 선택된 계정을 팀에서 제거 삭제 후 리스트 갱신  owner 의 경우 삭제 불가 → 체크박스가 없음  |  |
|  | 1.5 사용자 목록 로드 방식 | 사용자 수가 많을 경우 무한 스크롤 방식으로 추가 로딩 |  |
| 사용자 초대 | 2.1 직접 계정 추가 | 상단 `계정 추가` 버튼 클릭 시 이메일 입력창 노출 이메일 입력 시 MAIT 가입자 내에서 검색 정확한 이메일을 입력한 경우에만 해당 인원이 검색됨 계정 일치 시 검색 결과에 닉네임/이름/이메일 이 표시됨 검색 입력창 우측에 권한 선택 드롭다운(maker/player) 제공  클릭 시 각 역할 설명 확인 가능 이메일 입력 & 권한 선택 후 초대 버튼 클릭 초대 성공 시 해당 계정은 선택한 권한 리스트에 즉시 반영 초대 알림 초대된 사용자에게 알림 문구: "~~팀에 maker로 초대되었습니다." 알림 클릭 시 해당 팀의 문제 풀기 페이지로 이동 |  |
|  | 2.2 초대 링크 생성 | `초대 링크 생성` 버튼 클릭 시 모달창 설정 항목: 초대 권한 유형 (maker / player) 링크 유효기간 (1일 / 3일 / 7일) 승인 절차 유무 (O / X) player 선택 시 승인 절차는 무조건 `X`로 고정 |  |
|  | 2.3 초대 링크 생성 결과 | `생성하기` 클릭 시 초대 링크 발급 링크 텍스트 영역과 `복사하기` 버튼 노출 생성된 링크는 수정 불가 상태로 표시 |  |
|  | 2.4 초대 링크 관리 | 하단 `초대링크 관리` 영역에서 링크 목록 확인 가능 표시 항목: 권한 / 링크(복사 아이콘) / 종료 일시 / 삭제 아이콘 삭제 아이콘 클릭 시 삭제 확인 모달창 확인 클릭 시 즉시 삭제  |  |
|  | 2.5 승인 절차 동작 조건 | maker 초대의 경우, 승인 절차를 'O'로 설정 시 관리자의 수락 후만 팀에 등록 가능 player 초대의 경우 승인 절차 무조건 'X'로 설정되어 자동 승인 |  |
|  | 2.6 초대 링크 접속 화면 | 초대 링크로 접속 시 링크 상태 / 로그인 여부 / 초대 유형에 따라 화면이 달라짐 링크 만료 또는 비유효 버튼: 로그인하기 / 회원가입하기 링크 유효 + 로그인 전 버튼: 로그인하고 참여하기 / 회원가입 후 참여하기 로그인 시 승인 필요 없는 경우 → 팀 페이지로 이동 (문제 풀기) 승인 대기 상태의 경우 → 4번 페이지 안내를 모달창으로 안내  링크 유효 + 로그인 상태 + 승인 불필요 버튼: 참여하기 버튼 클릭 시 문제 풀기로 이동 링크 유효 + 로그인 상태 + 승인 필요 버튼: 확인 버튼 클릭 시 메인 페이지 또는 사용자가 이전에 머무르던 페이지로 이동 승인 완료 시 헤더 알림창에 완료 안내 알림 승인 거절된 경우 버튼: 홈으로 이동 |  |

## 2. (2/19) 카테고리 관리

| **구분** | **세부기능** | **상세설명** | **코멘트** |
| --- | --- | --- | --- |
| 카테고리 관리 | 1.1. 카테고리 추가(추가) | 카테고리명 중복 불가 '이미 존재하는 카테고리입니다.' 삭제한 카테고리명과 동일한 카테고리를 다시 생성할 때 삭제된 동일 이름 카테고리가 있습니다. 기존 카테고리를 복구할까요? |  |
|  | 1.2. 카테고리 삭제 | 카테고리를 관리페이지에서 삭제했는데 해당 카테고리를 이미 사용중인 문제셋이 있는 경우 → 대시보드에도 나와있을 거라는 뜻 **삭제 가능. 하지만 기존에 사용했던 문제셋이나 대시보드에서 없어지지는 않음. 그대신 삭제 이후 문제셋 생성 시 카테고리 후보에 해당 카테고리가 없음** |  |
|  | (참고) 문제 정보 입력 시 카테고리 선택 | 카테고리 드롭다운 형식으로 선택 카테고리 선택 개수 제한 없음 카테고리 리스트 내 선택 또는 검색하여 선택 선택한 카테고리 상단으로 이동(하단 리스트에서 없어짐) 존재하지 않는 카테고리 검색 시 직접 추가(생성) 가능 '카테고리 관리'와 연동 |  |
