### 버튼명

- `문제 만들기` 버튼명은 큰 이슈가 없는 것으로 보았다.
- 최종 생성 단계의 버튼명은 수정이 필요하다고 정리했다.
- 후보로 `문제 셋 생성하기?`, `문제 셋 저장하기`, `publish`, `등록`, `게시`, `공개` 등이 언급됐다.
- 버튼을 누르기 전 상태는 문제 또는 문제셋을 제작 중인 상태로 보았다.
- 버튼을 누른 후에는 문제셋 풀이 시작이 가능한, 풀기 전 상태로 보았다.
- 결론은 버튼명을 바꾸면서 함께 정해질 것으로 정리됐다.

### 문제 정보 입력, 문제 셋 생성 용어 정리

- `문제`, `문제셋` 관련 네이밍 자체가 없어서 헷갈린다는 문제가 제기됐다.
- `문제셋을 만든다`고 말할 때 정보 입력 단계와 최종 생성 단계가 나뉘므로 용어 정리가 필요하다.
- 결론은 버튼명 변경과 함께 용어가 같이 정해질 것으로 정리됐다.

## 6. AI의 요약

- 6월 16일 배포는 풀이결과 대시보드, 실시간 등수 정보 1차, 팀 간 이동 차단, 메인 반응형/애니메이션 이슈를 중심으로 진행 상황을 확인했다. 배포 관련 문서는 알파 반영 여부를 포함해 다음 날 오전 공유하고, 6월 19일 코테이토 사용을 위해 추가 기능 명세는 수요일까지 작성해 피드백받기로 했다.
- 문제 생성 흐름에서는 `문제 만들기` 자체는 큰 이슈가 없지만, 최종 생성 단계의 버튼명은 수정이 필요하다고 정리했다. 문제/문제셋 용어가 헷갈리는 원인은 네이밍 부재에 있고, 버튼을 누르기 전에는 문제 또는 문제셋 제작 중 상태, 누른 뒤에는 문제셋 풀이 시작이 가능한 상태로 구분해 용어와 버튼명을 함께 정하기로 했다.
- 풀이결과 대시보드와 등수 노출은 실제 데이터 기반 화면 구성이 필요하다고 확인했다. 더보기 후 접기 동작, 날짜/정답률 정렬 기준의 화면 정의, 실시간/학습 태그 노출, 문제 유형별 답안 영역 기준, 풀이 기록이 없는 경우의 표시까지 추가 정리가 필요하며, 백엔드에서는 득점자를 정답자에 포함시키는 로직 변경이 남았다.
- 어드민 페이지는 당장 사용자가 볼 기능보다는 GA/뷰저블로 잡히지 않는 데이터, 유저 관리, FE QA 전용 API를 제공하는 내부 지원 도구로 방향이 확인됐다. 백엔드 할일로 기획자에게 필요한 데이터 전달과 QA용 API 실행이 남았다.
- 파트별 공유에서는 프론트의 초 단위 차이, 참여자 카운트, 대시보드, 온보딩 테스트, 메인 애니메이션 이슈가 완료로 체크됐다. 디자인은 문제 입력 통일 UI는 완료했지만 등수 기획 UI와 대시보드 수정이 남았고, 기획은 버튼명·유형별 노출 기준·등수 노출 화면 구성을 이어가야 한다.
- 운영 항목으로 `mait.kr` 도메인이 2026년 6월 28일 만료 예정이라 팀비를 사용해 연장하기로 했고, 비용은 2년 약 44,000원 또는 3년 약 65,000원 중 2년 연장이 유력하게 논의됐다. 7월 4일 토요일 4시 대면 회의는 사당 쪽으로 예상했다.
