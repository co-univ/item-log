### 6. AI의 요약

- 5월 15일 사용에서는 별도 이슈가 없었고, 5월 12일 메인 배포 이슈는 별도 정리 페이지로 공유됐다. 이번 회의의 실제 기획 논의는 배포 후속 정리보다 문제 생성·카테고리·결과 화면의 다음 개선 범위를 정리하는 쪽에 집중됐다.
- 사용자 문의에서는 카테고리를 팀별로 자율 설정하는 해시태그/공통 범위 개념으로 설명했다. 학습모드는 실시간 모드가 자동 복사되는 흐름이 아니라 별도 풀이 방식이며, 실시간·학습으로 완료된 문제셋을 maker가 복습 모드로 넘길 수 있다는 점도 다시 정리됐다.
- 문제 생성 정보 입력 페이지는 MAIT-154 범위로 통일한다. 직접 제작에서도 자료 업로드·보충 설명 같은 필드는 노출하되 수정 불가 상태를 고려하고, 문제 유형 기본값은 각 유형 1개씩으로 정했다. 주제는 제목과 통합하고, 문제 풀이 대상은 보충 설명으로 흡수하는 방향이 논의됐다.
- 유형별 제출 현황은 MAIT-143의 후속으로 본다. 유형별 구조는 피그마에 확정되어 있지만, 퍼센트 바에서 90:10처럼 작은 영역에 텍스트를 넣기 어려운 케이스가 있어 비율·개수 노출 기준을 기획과 디자인이 추가로 정해야 한다.
- 온보딩은 MAIT-85와 연결해 디자인 검토를 이어간다. 블러 처리, 전후 이동 버튼, 실시간 랭킹 정보 조회 영역 확대가 논의됐고, 프론트는 실시간 랭킹 정보 영역을 검토한 뒤 공유하기로 했다.
- 카테고리 관리 UX는 별도 후속으로 남겼다. 수정 완료는 체크 아이콘으로 하고, 취소를 위해 체크/엑스 형태를 둔다. 목록은 한 번에 하나씩 다루며 최근 추가 항목은 아래로 정렬하는 방향으로 정리되어 신규 MAIT-155를 만들었다.
- 다음 배포 범위는 기존 Jira 이슈를 재사용한다. 개인 워크스페이스, 풀이결과 대시보드, 등수 정보, 알림, 팀 삭제, 어드민 페이지는 이미 활성 이슈가 있으며, 백엔드·프론트·디자인 파트의 할 일은 그 이슈들과 연결해 진행하는 흐름이다.
