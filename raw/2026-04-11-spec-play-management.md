# 문제관리_풀이관리

| **구분** | **세부기능** | **상세설명** | **의견** |
| --- | --- | --- | --- |
| 선착순 모드 | 1.0. 시작하기 버튼 | 시작하기를 누르면 진행 예정에서 진행 중으로 상태가 변경됨 시작하기를 안 누른 상태에선 모든 문제에 노출되며 시작하기를 눌러서 진행중으로 상태가 변경된 이후에는 종료하기 버튼으로 변경 |  |
|  | 1.1. 문제 영역 (질문, 답안) | 질문과 선지(w. 정답)가 모두 노출 객관식, 순서: 질문, 선지(정답도) 노출 주관식, 빈칸: 질문, 정답 노출 |  |
|  | 1.2 답안 추가 | 선지(정답)에 수정 아이콘: 연필 아이콘을 클릭 > 문제 정답 수정 > 완료 > 문제 정답이 업데이트 (아래의 내용대로 간다면 주관식과 빈칸 유형에만 연필아이콘-> 연필아이콘 대신 인정답안 추가 버튼? 아니면, 수정) 객관식: 답안 수정 불가 주관식: 인정답안 추가만 가능 순서: 정답(순서)만 변경 불가, 보기 추가 및 내욘 수정 불가 (답안 수정 불가) 빈칸: 빈칸 자체의 수정은 불가(빈칸을 추가하거나 빈칸의 위치를 바꾸는 것 불가), 빈칸 답안의 인정답안 추가만 가능  | (Figma 링크) 주관식과 빈칸 유형의 인정답안 추가만 가능함! |
|  | 1.3. 문제 제어 | 문제공개 토글: 사용자에게 문제화면이 공개됨 (off되어있는 경우, 문제 내용을 볼 수 없음) 제출허용 토글  on하면 랜덤 시간 내에 종 애니메이션이 노출되면서 '제출하기'가 활성화 off인 경우, '제출하기' 비활성화 문제공개가 되어야 제출허용이 될 수 있는 구조 |  |
|  | 1.4. player에 대한 정보와 player가 입력(선택)한 답안  | maker에 대한 정보: 실명+닉네임 노출(9자 제한, 넘을 시 말줄임) (1) 선착순: 먼저 제출한 사람부터 나열 (2) 정답자: 정답자를 나열 (선착순 기반) (3) 오답자: 오답자를 나열 (선착순 기반) maker가 입력(선택)한 답안 (디자인 확인 후 글자수 제한 필요) **무한 스크롤 방식** 리스트 내에서 정오답을 명확하게 표시 eg. 빨강, 파랑 |  |
|  | 1.5. 득점자 | 득점자 노출: 가장 빠르게 정답을 제출한 사람 1.1. 이후에 클릭 시 수정된 정답에 따라 득점자가 변경될 경우 득점자가 자동 업데이트 |  |
|  | 1.6. 진출자 선정 | '진출자 버튼' 클릭 시 새로운 페이지로 이동 player 관련 정보 선착순 기준 등수: 가장 빠르게 많은 정답을 제출한 사람 순으로 노출 공동 등수일 경우 가나다순 등수 앞에 체크박스 존재 직접 선택 하단 등수 입력을 통한 자동선택 (eg. 선착순 __등 까지 에서 3을 입력하면 1등부터 3등까지의 체크박스가 자동 선택됨  정답자 기준 등수: 선착순과 상관없이 문제 자체를 맞은 개수로 계산 동일한 등수의 사람이 많기 때문에 동일 등수는 리스트 나열이 아닌, 쉼표로 표시 다음단계 진출자 선정 영역 1 또는 2에서 선택한 player가 해당 영역에 표시됨 직접 입력도 가능: _이하은(웃긴 감자)_ 와 같이 정확히 입력한 경우에만 가능 '진출자 확정하기' → 확인 모달 (0000 진출자로 확정하시겠습니까?) 확인 시 풀이관리페이지로 이동(1.1. 이 있는 페이지) 취소 시 1.5. 페이지에 머문다 |  |
|  | 1.7. 문제 이동 | 상단에 있는 번호 클릭을 통한 이동 '이전문제', '다음문제'를 통한 문제 이동 |  |
|  | 1.8. 종료하기 버튼 | 시작하기를 한 이후에는 모든 문제에 종료하기 버튼 존재 종료하기 호버 시 '우승자 정보 player에게 공개하기'  종료하기 클릭 시 '우승자는 000입니다.  PLAYER 에게 결과를 공개하시겠습니까?' 팝업 노출  전송하기: player에게 우승자 알림창 노출, maker에게도 동일한 화면 노출 > 종료하기 >  복습상태로 이동 전 공개범위설정(기존 문제셋의 설정으로 체크되어 있으며 다른 상태로 변경 가능)  > 확인 > 복습으로 이동된 화면 노출 종료하기: 우승자 전송 없이 그냥 종료됨 > 복습상태로 이동 전 공개범위설정( 기존 문제셋의 설정으로 체크되어 있으며 다른 상태로 변경 가능) > 확인 > 복습으로 이동된 화면 노출 |  |
| 학습 모드 | 문제 풀이 시작  문제 종료 | 문제 풀이 시작 문제 시작 시 '시작하기' 버튼 클릭을 통해 문제 활성화 문제 종료 모든 인원이 풀기 완료 시 자동 '종료' 처리 문제 셋 생성 → 풀이 중 → 팀원 알파 새로 가입 → 기존 팀원 풀이 완료 한 경우, 종료처리 이전에 들어온 player까지 팀원으로 간주하여, 해당 인원까지 다 풀어야 자동 완료 처리 수동 종료도 가능 |  |
|  | 1.1. 문제 영역 (질문, 답안) | 질문과 선지(w. 정답)가 모두 노출 객관식, 순서: 질문, 선지(정답도) 노출 주관식, 빈칸: 질문, 정답 노출 선지(정답)에 수정 아이콘: 연필 아이콘을 클릭 > 문제 정답 수정 > 완료 > 문제 정답이 업데이트 |  |
|  | 1.2.답안 추가 | (선착순 모드 1.2와 동일한 정책) |  |
|  | 1.3. player에 대한 정보와 player가 입력(선택)한 답안  | maker에 대한 정보: 실명+닉네임 노출(9자 제한, 넘을 시 말줄임) (1) 선착순: 먼저 제출한 사람부터 나열 (2) 정답자: 정답자를 나열 (선착순 기반) (3) 오답자: 오답자를 나열 (선착순 기반) maker가 입력(선택)한 답안 (디자인 확인 후 글자수 제한 필요) **무한 스크롤 방식** |  |
|  | 1.4. 문제 이동 | 상단에 있는 번호 클릭을 통한 이동 '이전문제', '다음문제'를 통한 문제 이동 마지막 문제에서는 '다음문제'가 '종료하기'로 변경 종료하기 클릭  >  복습상태로 이동 전 공개범위설정(전체공개, 그룹공개, 비공개 중 선택)  > 확인 > 복습으로 이동된 화면 노출 |  |
