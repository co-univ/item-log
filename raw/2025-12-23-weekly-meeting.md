## 원문 스냅샷

### 운영 메모

- 연말 일정 때문에 화/수 분산 진행, 작업 인증은 디스코드 사용, 작업 내용은 별도 문서 정리, `디스코드 문의 채널 생성`, `JIRA 사용`이 공지됐다.
- `회고 개선 사항 시도`를 위한 안건 작성 요청도 남아 있다.

### 파트별 공유

- 백엔드는 `복습 정답 제출 API`, `AI 문제 생성이 간헐적으로 실패하던 이슈 수정`, `마지막 문제 조회 API 권한 이슈 수정`, `복습 - 제출한 정답에 뭐가 맞고 틀렸는지`, `복습 모드 문제 조회 시 정오답 노출 제거`, `문제 풀이 문제 노출 시 해설 필드 제거`, `JIRA 세팅 공유`, `주관식 정답 개수 데이터 오류 보정`, `API 권한 관리 draft 작성 완료`를 남겼다.

### 기획/문의 정리

- `재채점 관련 문의` 재문의와 답변 처리, `디스코드 문의 답변`, `문제풀기 기능명세서 업데이트`, `플로우차트/기능명세서-피그마 연결 형식`이 남았다.
- 디자인에서는 `og태그`가 완료 처리됐다.

## AI 후속 블록 스냅샷

### 6. AI의 요약

- 연말 배치 회의록은 기능 기획보다 운영 체계 정비와 복습 모드 세부 동작 점검이 더 강하게 드러난다.
- 제품 면에서는 복습 정답 제출 API가 이미 붙어 있었고, 그 다음 과제로 `어떤 답이 맞고 틀렸는지`, `복습 조회 시 정오답 노출 제거`, `문제 조회 시 해설 필드 제거`처럼 결과 노출 정책을 더 정교하게 손보려 했다.
- 동시에 `AI 문제 생성 간헐 실패 수정`과 `마지막 문제 조회 API 권한 이슈 수정`, `API 권한 관리 draft 작성 완료`가 한 묶음으로 보이면서, 이 시기엔 AI 연동 안정화와 권한 정책이 실제 장애/운영 이슈로 전환됐음을 알 수 있다.
- 디스코드 문의 채널, Jira 사용, 재채점 문의 응답 같은 운영 메모까지 묶여 있어, 제품 개발과 사용자 지원 프로세스를 같이 정비하던 연말 운영 로그 성격도 강하다.
