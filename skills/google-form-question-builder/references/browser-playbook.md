# Google Forms 브라우저 플레이북

브라우저 UI 자동화로 Form을 만들거나 갱신하고 화면 이미지를 넣을 때만 이 문서를 사용한다. Google UI의 접근성 이름을 우선 사용하고, 일회성 스냅샷 참조값이나 화면 좌표에 의존하지 않는다.

## 질문 찾기

- 질문 편집기는 보통 `[role="textbox"][aria-label="질문"]`으로 찾을 수 있다.
- 텍스트를 정규화한 뒤 정확한 질문 문구로 대상을 찾는다.
- 질문 위치만으로 매칭하지 않는다. 질문 목록이 바뀌면 위치가 쉽게 어긋난다.
- 질문 카드가 필요하면 질문 편집기에서 가장 가까운 카드 조상으로 이동한다. 현재 UI에서는 `ancestor::div[@jsname="CkZkVb"][1]`가 동작하지만, 구조가 바뀌면 질문 카드 안의 `필수`, `질문 삭제`, `인라인 이미지 추가` 컨트롤을 기준으로 가장 가까운 공통 조상을 다시 찾는다.

## 객관식 보기

- 객관식 보기 입력은 현재 `input[aria-label="옵션 값"]`으로 확인할 수 있다.
- `옵션 1` 같은 기본값이 남지 않았는지 전체 Form에서 검사한다.
- 값을 채운 뒤 포커스를 이동해 변경 이벤트와 자동 저장이 발생하게 한다.
- 질문 카드가 비활성 상태면 편집 컨트롤이 숨겨질 수 있으므로 질문을 먼저 클릭한다.

## 인라인 이미지 업로드

1. 질문을 클릭해 카드 편집 상태를 연다.
2. `인라인 이미지 추가` 버튼을 누른다.
3. 표시된 Google Picker iframe을 찾는다. URL은 일반적으로 `https://docs.google.com/picker?`로 시작한다.
4. `업로드` 옵션을 누른 뒤 `input[type=file]`에 절대 경로의 캡처 파일을 설정한다.
5. 카드 안에서 실제 `src`가 있는 이미지가 생길 때까지 기다린다.

빈 이미지 플레이스홀더가 모든 카드에 존재할 수 있다. 단순 `img` 개수로 업로드 성공을 판단하지 말고 `img[src]`, 가능하면 `src`에 `forms-images`가 포함된 실제 서버 이미지를 확인한다.

이미지 선택기 iframe이 여러 개 남아 있으면 화면에 보이는 마지막 Picker frame을 사용한다. 업로드 완료 후 Picker와 대화상자가 닫혔는지 확인한다.

## 캡처

- 로그인 후 대상 기능을 읽기 전용으로 탐색한다.
- 개인정보가 있으면 캡처 직전 DOM의 표시 텍스트만 `참여자 A`, `닉네임 A`처럼 바꾼다. 입력값이나 네트워크 요청을 변경하지 않는다.
- 전체 화면보다 질문의 판단 문맥을 보여주는 영역을 우선한다.
- 임시 파일 이름에는 기능을 드러내는 slug를 사용한다.

## 저장과 게시 확인

- 상단 저장 상태의 접근성 설명에서 `드라이브에서 모든 변경사항이 저장되었습니다`를 확인한다.
- 미게시 Form에는 `게시` 버튼이 보여야 하며 `게시됨` 설정 버튼이 없어야 한다.
- 게시 취소가 필요하면 상단 `더보기` 메뉴의 `양식 게시 취소`를 사용하고 확인 대화상자에서 다시 확인한다.
- 기존 Form이 게시 상태라면 사용자의 요청 없이 게시 취소하지 않는다.

## 최종 DOM 검사

다음 값을 한 번에 수집해 검증한다.

- 모든 질문 제목과 개수
- 빈 제목과 중복 제목
- 모든 `옵션 값`
- `필수` 체크박스의 활성 개수
- 실제 Google Forms 이미지가 있는 질문 제목과 이미지 개수
- 저장 상태와 게시 상태
- 열려 있는 대화상자 개수

검증 결과가 예상과 다르면 완료라고 보고하지 말고 해당 질문만 다시 수정한다.
