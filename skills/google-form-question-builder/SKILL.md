# Google Form 질문 만들기

질문 소스를 기준으로 Google Form을 생성하거나 안전하게 갱신하고, 필요한 제품 화면을 캡처해 질문별 참고 이미지로 삽입한다. 브라우저 자동화가 필요하면 먼저 [브라우저 플레이북](references/browser-playbook.md)을 읽는다.

## 작업 모드 결정

1. 질문 소스와 대상 Form을 식별한다.
   - 대상 Form이 없으면 새 Form을 만든다.
   - 편집 링크나 이전 작업의 Form이 있으면 기존 Form 갱신으로 처리한다.
   - 어느 Form을 바꿔야 하는지 안전하게 특정할 수 없을 때만 사용자에게 묻는다.
2. 소스를 끝까지 읽고 섹션, 질문 제목, 질문 유형, 보기, 필수 여부, 필요한 화면 단서를 구조화한다.
3. 기존 Form 갱신이면 현재 섹션·질문·보기·이미지·게시 상태를 먼저 읽고 소스와 차이를 계산한다.
4. 변경 예정 항목과 캡처가 필요한 화면을 짧게 알리고 진행한다.

## 질문 소스 반영

소스를 질문 문구와 순서의 기준으로 사용한다. 임의로 질문을 추가하거나 의미를 바꾸지 않는다.

- 자유 서술 질문은 `장문형`을 기본으로 사용한다.
- 짧은 식별값이나 한 줄 답변은 `단답형`을 사용한다.
- 예/아니오/모름처럼 선택지가 명시된 질문은 `객관식`으로 만든다.
- 소스에 섹션이 있으면 Form 섹션으로 보존한다.
- 필수 여부가 명시되지 않으면 선택 문항으로 둔다.
- 제목 접미사(예: `- 테스트`)는 사용자가 요청했을 때만 붙인다.

기존 Form을 갱신할 때 다음 규칙을 적용한다.

- 제목만 동일한지 보지 말고 섹션과 의미를 함께 비교한다.
- 변경되지 않은 질문과 여전히 유효한 이미지는 유지한다.
- 바뀐 질문의 기존 이미지가 새 질문에도 맞는지 확인하고, 맞지 않으면 새로 캡처해 교체한다.
- 소스에서 사라진 질문의 삭제, 질문 유형 변경, 대규모 재정렬은 응답 데이터에 영향을 줄 수 있다. Form에 응답이 있거나 게시 중이면 사용자 확인 없이 실행하지 않는다.
- 응답이 없고 미게시 상태이며 사용자가 소스 동기화를 명시했다면 소스와 일치하도록 수정한다.

## 로그인 경계 처리

Google 또는 화면 캡처 대상 서비스에 인증이 필요하면 로그인 화면까지만 이동한 뒤 사용자 입력을 기다린다.

- 비밀번호, Basic Auth, OTP, 보안키는 사용자가 브라우저에 직접 입력하게 한다.
- 채팅으로 자격 증명을 요청하거나 브라우저에서 값을 읽고 출력하지 않는다.
- 사용자가 `로그인 완료`라고 알리면 현재 로그인 상태를 확인하고 작업을 재개한다.
- 세션이 만료되면 같은 방식으로 다시 로그인만 요청한다.
- CAPTCHA나 계정 복구 절차를 우회하지 않는다.

## 관련 화면 캡처

질문별로 답변에 실제 도움이 되는 화면을 매핑한 뒤 캡처한다.

1. 질문이 묻는 기능과 가장 가까운 목록, 상세, 편집, 관리, 결과 화면을 선택한다.
2. 읽기 전용 탐색을 기본으로 한다. 모달 열기, 탭 전환, 필터 변경처럼 되돌릴 수 있는 조작은 허용하되 실제 데이터 생성·수정·삭제·진행 상태 변경은 사용자 승인 없이 하지 않는다.
3. `관련 화면을 모두`라는 요청은 서로 다른 핵심 흐름을 빠짐없이 포함한다는 뜻으로 해석한다. 같은 의미의 화면 변형을 무한히 캡처하지 않는다.
4. 참여자 이름, 닉네임, 이메일, 전화번호, 토큰 등 개인정보와 비밀값을 제거한다.
   - 가능하면 캡처 직전 브라우저 DOM에서 표시 문자열만 임시 익명화한다.
   - DOM 익명화가 어렵다면 안전한 영역만 자르거나 이미지 편집으로 가린다.
   - 백엔드 데이터는 수정하지 않는다.
5. 각 캡처에 기능을 식별할 수 있는 충분한 문맥을 남기고 불필요한 개인 정보 영역은 제외한다.
6. 캡처 파일은 작업 전용 임시 디렉터리에 두고 Form 업로드와 검증 후 삭제한다.

## Google Form에 이미지 삽입

- 질문 카드의 인라인 이미지 기능으로 해당 질문 바로 아래에 이미지를 넣는다.
- 한 이미지가 여러 질문에 관련돼도 가장 직접적인 질문에 우선 배치한다. 중복 첨부가 실제 이해에 필요할 때만 반복한다.
- 이미지와 질문의 매핑을 기록해 최종 검증에 사용한다.
- 업로드 직후 로컬 미리보기만 보지 말고 Google Forms 이미지 URL이 생성됐는지 확인한다.

## 게시 상태와 안전

- 새 Form은 기본적으로 미게시 상태로 둔다.
- 기존 Form의 게시 상태와 응답 수집 상태를 보존한다.
- 작업 중 실수로 게시됐으면 응답 수집을 중지한 뒤 `양식 게시 취소`로 원래 상태를 복원한다.
- 사용자가 명시적으로 요청하지 않으면 응답자 링크를 배포하거나 다른 사람에게 공유하지 않는다.

## 완료 검증

다음을 모두 확인한 뒤 완료로 보고한다.

- 질문 수와 섹션 수가 소스와 일치한다.
- 빈 질문과 중복 질문이 없다.
- 각 질문 유형, 객관식 보기, 필수 여부가 의도와 일치한다.
- 계획한 모든 질문에 이미지가 하나씩 들어갔고 잘못 매핑된 이미지가 없다.
- 개인정보가 캡처에 남지 않았다.
- 모든 변경사항이 Google Drive에 저장됐다.
- 열린 이미지 선택기나 확인 대화상자가 없다.
- 게시 상태가 작업 전 상태 또는 사용자가 요청한 상태다.
- 임시 캡처 파일을 삭제했다.

최종 답변에는 Form 편집 링크, 질문·섹션·이미지 수, 게시 상태, 중요한 수정 사항을 간단히 적는다.
