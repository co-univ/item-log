# (player뷰) 문제 풀기 명세서

player 측 풀이 화면. 실시간 / 학습 / 복습 3가지 모드 각각의 동작·이탈 처리·채점 방식 정의.

내부 히스토리: 12/30(1.4 오답 / 1.5), 3/6(2.1, 2.5), 4/11(3번 문제셋 상태) 업데이트.

## 핵심 요약

### 실시간 모드 (선착순 짝)
- **maker 권한 의존**: `문제공개 ON` → 문제 풀이 가능, `제출허용 ON` → 종 애니메이션 + 제출 버튼 활성화.
- **대기화면**: 문제공개 OFF 상태일 때.
- **다음 진출자 화면**: maker가 진출자 선정 시.
- **재제출 정책**: 정답이면 답안 수정 불가. 오답이면 재제출 가능 (단, 풀이관리에서 정답자·오답자 모두에 표기되는 이슈는 미결).
- **이탈 복귀**: 데이터 보존. 진행 중 문제로 자동 이동. 종료 후 재진입해도 데이터 유지.

### 학습 모드
- **문제셋 상태 (4/11 추가)**: 풀이 전 / 풀이 중(자동저장) / 풀이 완료(채점하기 후).
- **자유 이동**: 문제 번호 클릭으로 이동. 푼 문제 / 안 푼 문제 시각 구분.
- **"문제를 풀었다" 기준** — 객관식: 1개 이상 선택 / 주관식·빈칸: 답안 영역 모두 채움 / 순서: 모두 채움.
- **제출 단위는 문제셋 전체**. 제출 후 채점 뷰로 이동, 정/오답 + 풀이 + 해설 한꺼번에. **재풀이 불가**.
- **자동저장**: 다른 문제로 이동 시 이전 문제 저장.
- **이탈 처리**: 풀이 기록 일부 유실 가능 팝업 → 확인 시 저장된 부분만.

### 복습 모드
- **자유 이동** + 정답/해설 토글 흐름.
- To-be 정책: 제출 → 채점 → 다시풀기 / 해설보기 / 해설닫기.
- **여러 번 풀이 가능** (자유 노트). 단순 1회 풀이는 복습 의미와 어긋남.
- **이탈 복귀**: 24시간 내 재진입 → 마지막 푼 문제 + 정답 OFF + 해설 OFF. 24시간 초과 → 1번 문제 + 정답·해설 OFF.

### 별첨 — 복습 모드 정답 표시 정책
- 단답(객관식 1개 / 주관식 1개 / 빈칸 1개 / 순서): 틀리면 빨강, 맞으면 초록 (현 동작 유지).
- 다답(2개 이상): **사용자가 선택·작성한 답 단위로 부분 채점** — 맞은 건 초록, 틀린 건 빨강.
- 실시간/학습 모드는 현 동작 유지 (하나라도 틀리면 오답).
