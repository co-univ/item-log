# 2025-11-16 주간 회의록

11월 셋째 주 회의는 세부 기능 하나를 깊게 파기보다, 출시 전 남은 범위를 다시 구조화한 회의다. 문제 생성, 대시보드, 팀관리, 학습/복습, 소셜 로그인, API 권한, 검색, 온보딩이 한 번에 테이블 위에 올라왔다. 이는 문서와 구현이 병렬로 진행되던 제품이 한 번 전체 우선순위를 다시 맞춰야 하는 시점에 들어섰음을 보여준다. 여러 축이 동시에 남아 있었다는 점 자체가 당시 제품의 리스크 지형이다.

## 핵심 takeaway

- 이 회의는 개별 명세 확정보다 남은 작업 묶음을 다시 나누고 순서를 맞추는 성격이 강했다. 근거: [raw/2025-11-16-weekly-meeting.md](../../raw/2025-11-16-weekly-meeting.md) `§원문 스냅샷`, `§AI 후속 블록 스냅샷`. 영향: [문제 검색](../features/problem-search.md), [문제 풀기 (player 측)](../features/play-player.md), [팀 사용자 초대](../features/team-invitation.md)
- 대시보드와 AI 생성, API 권한, 온보딩이 같은 회의에서 함께 남아 있었다는 점은 출시 직전 완성도 확보가 핵심 과제였음을 보여준다. 근거: [raw/2025-11-16-weekly-meeting.md](../../raw/2025-11-16-weekly-meeting.md) `§AI 후속 블록 스냅샷`. 영향: [풀이결과 대시보드](../features/result-dashboard.md), [AI 문제 생성](../features/ai-question-generation.md), [로그인 / 회원가입 / 마이페이지](../features/auth-and-account.md)
- 학습/복습과 팀관리, 검색은 서로 분리된 기능이 아니라 같은 릴리스 범위 안에서 우선순위를 조정받고 있었다. 근거: [raw/2025-11-16-weekly-meeting.md](../../raw/2025-11-16-weekly-meeting.md) `§원문 스냅샷`. 영향: [풀이관리 (maker 측)](../features/play-management.md), [팀 멤버 권한 관리](../features/team-member-management.md)
