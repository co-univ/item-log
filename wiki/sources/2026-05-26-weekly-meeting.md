# 2026-05-26 주간 회의록

6월 2일 배포를 앞두고 개인 워크스페이스, 풀이결과 대시보드 2차, 알림, 팀 삭제, 카테고리 UI, 문제 카드 UI를 점검한 주간 회의록이다. 개인 워크스페이스는 자동 생성, 기존 유저 마이그레이션, 실시간 문제셋 생성 차단, 초대링크 차단 등 핵심 범위가 완료 체크됐다. 복습모드 전환 후 문제별 제출자 목록을 볼 수 없다는 사용자 문의를 계기로, 종료된 문제셋에서 maker/player별로 어떤 정보와 동선을 제공할지 별도 정책 정리가 필요해졌다. 풀이결과 대시보드 2차에서는 player의 완료 후 상세 대시보드 조회 가능 여부, 오답률 실시간 업데이트, 오답률 산식과 반올림 기준이 정리됐다. 등수정보는 실제 백엔드 데이터를 먼저 받아 와이어프레임을 재구성하기로 했고, 문제 카드 UI는 상태 기반 칸반/더보기 액션/카테고리 미노출 방향으로 후속 티켓이 분리됐다.

## 핵심 Takeaways

- 6월 2일 배포는 5월 31일까지 알파 반영하고 6월 1일 전체 QA를 진행하는 일정으로 정리됐다. 일부 알림 기능은 6월 16일 추가 배포로 밀렸고, 알림은 백엔드 우선 진행 후 프론트 반영을 추후 진행한다. 근거: `raw/2026-05-26-weekly-meeting.md` §공지사항.
- 개인 워크스페이스는 회원 가입 시 자동 생성, 기존 유저 자동 생성, 실시간 문제셋 생성 차단, 초대링크 생성 차단이 완료 체크됐다. 백엔드 후속으로 팀명 수정 API 확인, 마이그레이션 잡, 기본 생성 워크스페이스 명칭을 닉네임 기준으로 바꾸는 작업도 완료 체크됐다. 근거: `raw/2026-05-26-weekly-meeting.md` §2. 파트별 작업 내용 공유, §4. 할일 정리.
- 복습모드 전환 후 문제별 제출자 목록을 볼 수 없다는 사용자 문의는 단순 복구가 아니라 종료된 문제셋의 maker/player별 정보 노출 정책 문제로 분리됐다. 신규 Jira `MAIT-156`으로 maker/player 동선 정리를 추적한다. 근거: `raw/2026-05-26-weekly-meeting.md` §1. 지난 기간 발생했던 이슈 공유, §7. 할일 티켓 정리.
- player가 문제셋 풀이를 완료했지만 모든 팀원이 풀지 않았거나 관리자가 학습모드를 종료하지 않은 상태에서도 상세 대시보드를 볼 수 있다. 다만 오답률만 실시간으로 업데이트되며, 실시간/학습모드 player의 `풀이 완료` 상태에는 `통계확인` 버튼을 추가해야 한다. 근거: `raw/2026-05-26-weekly-meeting.md` §[PLAYER] 완료된 문제 셋 -> 상세 대시보드 조회 조건.
- 오답률은 `최초 제출 시 오답인 유저 / 해당 문제 제출 유저 수 * 100`으로 계산하고 소수 둘째 자리에서 반올림한다. 근거: `raw/2026-05-26-weekly-meeting.md` §오답률 산출 방식 정리.
- 등수정보 노출 기준은 실제 백엔드 문제 데이터를 기반으로 재논의한다. 기본 방향은 정답률 비례와 최대 영역 40%지만, 상위 N개 안에 정답이 없는 경우와 객관식 복수정답 그룹화 기준은 추가 명세가 필요하다. 근거: `raw/2026-05-26-weekly-meeting.md` §등수정보.
- 문제 카드 UI는 상태 기반 칸반, 제목/날짜 중심 카드, 더보기 메뉴 액션, 카테고리 미노출 방향으로 논의됐다. `문제관리`를 `팀관리` 하위로 옮기는 안은 관리자 메뉴를 모으는 장점과 정보구조상 부적절하다는 반대 의견이 함께 남았다. 신규 Jira `MAIT-157`로 후속 정리한다. 근거: `raw/2026-05-26-weekly-meeting.md` §문제 카드 UI 개선점 논의, §7. 할일 티켓 정리.
- 팀 삭제는 기존 `MAIT-82`에서 백필 후속으로 다시 확인됐다. 근거: `raw/2026-05-26-weekly-meeting.md` §4. 할일 정리, §7. 할일 티켓 정리.

## 미해결 / 후속 확인

- 종료된 문제셋에서 maker/player가 각각 볼 정보, 진입점, 권한 경계를 `MAIT-156`에서 정리해야 한다.
- player 완료 후 상세 대시보드 조회가 팀 학습모드에 한정되는지, 실시간/학습 완료 상태 전반에 적용되는지 최종 명세 확인이 필요하다.
- 오답률은 제출한 유저를 분모로 삼는 것으로 정리됐지만, 미제출자를 어떤 지표에서 별도로 다룰지는 아직 없다.
- 등수정보의 상위 N개 안에 정답이 없는 경우, 객관식 복수정답 그룹화, 유형별 노출 기준은 실제 데이터 기반으로 재설계해야 한다.
- 문제관리 정보구조는 독립 메뉴 유지와 팀관리 하위 편입 사이의 장단점이 남아 있다.
