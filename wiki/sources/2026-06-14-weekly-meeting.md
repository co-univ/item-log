# 2026-06-14 주간 회의록

6월 16일 배포와 6월 19일 코테이토 사용을 앞두고 배포 문서 공유, 추가 기능 명세 작성, 대시보드와 등수정보 후속을 맞춘 주간 회의록이다. 핵심 논의는 문제/문제셋 생성 용어와 최종 생성 버튼명, 풀이결과 대시보드의 정렬·태그·답안 영역 기준, 실제 데이터 기반 등수 노출 화면, 어드민 페이지의 내부 지원 목적에 집중됐다. 파트별로는 프론트의 시간 차이/참여자 수/대시보드/온보딩 테스트가 완료로 체크됐고, 디자인과 기획에는 등수 UI와 유형별 기준 명세가 남았다. 백엔드는 대시보드의 득점자 정답 처리, 어드민 데이터 전달, QA용 API 실행, 팀 삭제, 알림 초안, 문제 생성 UI 변경 관련 API 수정이 후속으로 남았다. Notion 티켓 정리에서는 신규 생성 없이 기존 `MAIT-64`, `MAIT-82`, `MAIT-126`, `MAIT-141`, `MAIT-143`, `MAIT-147`, `MAIT-148`, `MAIT-149`, `MAIT-154`를 재사용하는 것으로 기록됐다.

## 핵심 Takeaways

- 6월 16일 배포는 풀이결과 대시보드, 실시간 등수 정보 1차, 팀 간 이동 차단, 메인 반응형/애니메이션 이슈 중심으로 진행 상황을 확인했다. 배포 관련 문서는 알파 반영 여부를 포함해 다음 날 오전 공유하기로 했다. 근거: `raw/2026-06-14-weekly-meeting.md` §공지사항, §0616 배포 아이템 정리.
- 6월 19일 코테이토 사용을 전제로, 추가 기능 명세는 수요일까지 작성 후 공유하고 피드백을 요청하기로 했다. 근거: `raw/2026-06-14-weekly-meeting.md` §6월 19일 코테이토 사용.
- 문제 생성 흐름에서는 `문제 만들기` 자체는 큰 이슈가 없지만, 최종 생성 단계의 버튼명은 수정이 필요하다고 정리했다. `문제셋을 만든다`가 정보 입력과 최종 생성 중 어느 단계를 뜻하는지 모호해, 버튼명 변경과 함께 용어를 정리하기로 했다. 근거: `raw/2026-06-14-weekly-meeting.md` §버튼명, §문제 정보 입력, 문제 셋 생성 용어 정리.
- 풀이결과 대시보드는 더보기 후 접기, 날짜/정답률 정렬 기준, 실시간/학습 태그 노출, 문제 유형별 답안 영역 기준, 풀이 기록이 없는 경우의 표시를 추가로 정의해야 한다. 백엔드는 득점자를 정답자에 포함시키는 로직 변경이 남았다. 근거: `raw/2026-06-14-weekly-meeting.md` §0616 배포 아이템 정리, §4. 할일 정리.
- 등수정보는 실제 데이터 기반 화면 구성과 문제 유형별 기준 명세가 계속 후속으로 남았다. 시간 차이와 참여자 수 노출은 프론트 완료로 체크됐지만, 기획/디자인 기준은 아직 이어가야 한다. 근거: `raw/2026-06-14-weekly-meeting.md` §2. 파트별 작업 내용 공유, §4. 할일 정리.
- 어드민 페이지는 당장 사용자에게 보이는 기능보다 GA/뷰저블로 잡기 어려운 데이터, 유저 관리, FE QA 전용 API를 제공하는 내부 지원 도구로 방향이 확인됐다. 근거: `raw/2026-06-14-weekly-meeting.md` §백엔드, §4. 할일 정리.
- 7월 1일 배포/후속 범위로 팀 삭제, 알림 초안, 문제 생성 UI 변경 관련 API 수정이 백엔드 할 일에 남았다. 근거: `raw/2026-06-14-weekly-meeting.md` §4. 할일 정리.
- `mait.kr` 도메인이 2026-06-28 만료 예정이라 팀비를 사용해 연장하기로 했고, 2년 약 44,000원 또는 3년 약 65,000원 중 2년 연장이 유력하게 논의됐다. 근거: `raw/2026-06-14-weekly-meeting.md` §도메인 연장 비용.

## 미해결 / 후속 확인

- 최종 생성 버튼명과 문제/문제셋 용어의 확정 문구가 아직 없다.
- 대시보드의 날짜/정답률 정렬 기준, 태그 노출, 답안 영역 유형별 기준, 풀이 기록 없음 상태를 명세/디자인에 반영해야 한다.
- 득점자를 정답자로 포함하는 로직 변경이 기존 통계 산식과 오답률 산식에 어떤 영향을 주는지 확인해야 한다.
- 등수정보의 실제 데이터 기반 화면과 문제 유형별 기준 명세가 계속 필요하다.
- 어드민 페이지에서 기획자에게 전달할 데이터 범위와 FE QA용 API 범위가 아직 구체화되지 않았다.
- 이번 Notion 후속 처리 중 Jira 댓글 작성은 Atlassian 인증 오류로 완료하지 못했고, Notion에는 기존 티켓 재사용 기준만 기록됐다.
