# 2026-05-31 주간 회의록

6월 2일 배포 직전 알파/QA 상황과 다음 배포 일정을 점검한 주간 회의록이다. 핵심 제품 논의는 풀이결과 대시보드 2차, 개인 워크스페이스 종료/통계 전환, 카테고리 영역 단일화, 실시간 등수 노출, 온보딩과 문제 입력 UI 후속에 집중됐다. 대시보드에서는 학습모드 문제셋의 우승자 미노출, 사용자가 풀지 않은 문제셋의 통계 표시, 정답 통계 기준이 새로 정리됐다. 개인 워크스페이스는 풀이 시작/종료 기능이 없더라도 완료에 따른 자동 종료와 통계 전환은 유지하는 것으로 확인됐다. Jira 후속은 신규 생성 없이 기존 `MAIT-147`, `MAIT-149`, `MAIT-155`, `MAIT-85`, `MAIT-141`, `MAIT-157`, `MAIT-106`에 회의 코멘트를 추가했다.

## 핵심 Takeaways

- 6월 2일 밤 10시 배포를 앞두고 알파/QA 체크를 진행하며, 이슈는 카카오톡 또는 디스코드 미팅으로 수정 가능하다고 정리했다. 다음 일정은 6월 19일 사용, 7월 3일 배포 후보, 7월 4일 16시 회의 후보로 언급됐다. 근거: `raw/2026-05-31-weekly-meeting.md` §공지사항.
- 풀이결과 대시보드 2차에서 학습모드 문제셋은 우승자 영역이 없으므로 우승자 정보를 노출하지 않는 방향으로 정리했다. 문제셋 목록은 풀이 완료된 문제셋 범위에서 사용자가 직접 풀지 않은 문제셋도 포함한다. 근거: `raw/2026-05-31-weekly-meeting.md` §[대시보드2차] 문제 셋 노출 정보 문의.
- 사용자가 풀지 않은 문제셋의 대시보드 정보는 `내 정답률: - / 전체 평균 정답률`, 우리팀 랭킹과 오답률 Top3 유지, 점수 0점, 전부 틀린 문제로 반영하는 방향으로 기록됐다. 근거: `raw/2026-05-31-weekly-meeting.md` §[대시보드2차] 문제 셋 노출 정보 문의.
- 통계 정답 기준은 득점자라면 n번째 제출이어도 정답으로 카운트하고, 득점자가 아닌 경우 최초 제출 기록 기준으로 정답/오답을 처리한다. 근거: `raw/2026-05-31-weekly-meeting.md` §정답 데이터 기준이 첫번째 제출 기준인 경우 발생하는 모호함 문의.
- 개인 워크스페이스에서는 시작/종료 기능을 제외하더라도 문제 풀이 완료에 따른 자동 종료는 유지한다. Maker 수동 종료는 불가능하고, 풀이 완료 후 통계로 전환하며 상세 대시보드에서는 오답률을 노출하지 않는다. 근거: `raw/2026-05-31-weekly-meeting.md` §Discord 문의: 개인 워크스페이스 종료 처리.
- 카테고리/세부 과목처럼 보이던 두 영역은 실제 데이터 기준 `카테고리 이름` 하나로 합치고, 클릭 시 별도 이동은 하지 않는다. 근거: `raw/2026-05-31-weekly-meeting.md` §Discord 문의: 대시보드/카테고리 영역.
- 실시간 등수 노출은 참여자 수가 계속 표시되는 전제에서 문제 영역과 겹치는 문제가 있어 플로팅 버튼, 참여 인원 수, 참여자 닉네임, 이모지 노출을 대안으로 논의했다. 텍스트와 숫자가 커져 선지를 가릴 수 있다는 우려가 남았다. 근거: `raw/2026-05-31-weekly-meeting.md` §등수 노출.
- 온보딩은 앞/뒤 이동, 다시 보지 않기, 닫기, 순서 기능과 추가 플로우 팝업 처리가 후속으로 남았다. 문제입력 통일 UI, 등수 노출 디자인, 문제카드 UI 개선 고민도 계속 진행 중이다. 근거: `raw/2026-05-31-weekly-meeting.md` §2. 파트별 작업 내용 공유, §4. 할일 정리.

## 미해결 / 후속 확인

- 학습모드 대시보드에서 우승자 미노출을 카드 UI/상세 UI 어디까지 적용할지 최종 디자인 반영이 필요하다.
- 사용자가 풀지 않은 문제셋을 전부 틀린 문제로 반영하는 정책이 정답률/오답률/랭킹 산식 전체에 어떤 영향을 주는지 구현 기준을 확인해야 한다.
- 통계 정답 기준의 `득점자` 판정이 실시간 모드의 첫 정답자만 의미하는지, 문제 유형별 동점/복수 정답 상황에서 어떻게 적용되는지 세부 기준이 필요하다.
- 등수 노출 플로팅 버튼은 참여자 수, 닉네임, 이모지를 포함할 때 선지 영역을 가리지 않는 최종 디자인 검증이 필요하다.
- 온보딩 추가 플로우 팝업, 문제입력 통일 UI, 문제카드 UI 개선은 기존 Jira에서 계속 추적한다.
