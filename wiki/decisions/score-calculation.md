# 점수 산정 정책

풀이결과 대시보드 상세 화면의 점수 표기 정책.

## 표시 형식

`00점 (00/00)` — 점수 / (맞은 문제 수 / 전체 문제 수)

## 핵심 규칙

**분모는 항상 "문제셋의 전체 문제 수"이다.** 풀이 도중 탈락하여 뒷 문제를 풀지 못했더라도, 분모는 줄지 않고 못 푼 문제는 **오답 처리**.

### 예시

10문제 중 5번 문제에서 탈락한 player:
- 분자: 1~4번 중 맞은 개수 (예: 4)
- 분모: 10
- 결과: **40점 (4/10)**

## 의도 / 영향

- 탈락 시 분모를 줄이지 않음으로써 **완주 여부가 점수에 반영**되는 구조.
- player 간 비교를 항상 같은 분모(전체 문제 수)에서 하므로 공정함.
- 단점: 탈락 메커니즘이 강한 모드(선착순)에서는 평균 점수가 낮게 형성될 수 있음.

## 안 푼 문제의 처리

[features/result-dashboard.md](../features/result-dashboard.md) 2.4 — 안 푼 문제는 **오답 표시 + 정답·풀이 노출**. 결과 대시보드에서는 단순 확인 뷰이므로 다시 풀 수 없음.

## 미해결

- 학습 모드(전체 풀이가 가능한 모드)에서도 "탈락" 개념이 있는지 — 현재 명세는 선착순 모드 탈락만 가정.
- 인정답안 추가로 사후에 정답이 바뀌었을 때 점수 재계산 흐름.
