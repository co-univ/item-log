# 임시저장 / 자동저장 정책

문제셋 편집 중 데이터 보존을 위해 **수동 임시저장 + 자동저장** 2중 정책 채택.

## 임시저장 (수동, 3.10)

- 버튼은 두 가지 역할:
  1. 첫 임시저장 시 — 문제셋 **제목** 입력 + **주제** 자동 가져오기 (정보 입력 단계 값).
  2. 두 번째 액션부터는 단순 "저장".
- **활성화 규칙**: 변경 사항 없으면 비활성화, 생기면 활성화.
- 디자인 변경: 텍스트 없이 아이콘만.

## 자동저장 (3.11)

- **시작 시점**: 문제 편집 시점부터 즉시 활성화.
- **주기**: 변경 발생 후 **1분 뒤** 자동저장.
- **생성하기 미실행 + 화면 이탈 시**:
  - 자동 저장된 내용까지만 보존.
  - 메인 화면에 `임시 저장본n` 형식으로 노출.

## 디폴트 명명

생성하기 또는 임시저장조차 안 한 채 빠져나간 셋은 `문제셋1`, `문제셋2`... 으로 메인에 노출 (1.1).

## 왜 이중 저장인가

명세에 명시되진 않았지만 추론:
- 자동저장만으로는 의도적인 "버전 마커" 역할이 약함.
- 임시저장은 사용자가 명시적으로 "여기까지가 의미 있는 상태"라고 마킹하는 용도.
- 자동저장은 사고 방지(브라우저 닫힘, 네트워크 끊김 등) 안전망.

> 위 추론은 명세에 직접 명시된 내용이 아니므로, 의사결정 근거가 더 정리되면 업데이트 필요.
