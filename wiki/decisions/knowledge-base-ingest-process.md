# 지식 베이스 ingest 운영 원칙

회의에서 논의한 아이템의 히스토리와 마지막 결론을 잃어버리지 않기 위해, raw 회의록과 기능 명세 업데이트를 지식 베이스에 명시적으로 반영해야 한다.

## 결정 / 관찰

- 아이템별 논의 히스토리를 팀이 기억하지 못하는 문제가 있다.
- 문서 구조, 회의록 정리 등 여러 시도를 했지만 마지막 결론을 찾기 어렵다는 문제가 남았다.
- 원본 문서가 제대로 작성되지 않으면 지식 베이스도 부족할 수밖에 없다.
- 업데이트된 기능 명세서를 알려주지 않으면 레포에 정리하는 작업이 불가능하다.
- 디스코드 스레드에 논의된 내용은 아직 반영되어 있지 않다.

근거: [2026-04-26 주간 회의록](../sources/2026-04-26-weekly-meeting.md).

## 운영 의미

- 회의록은 `raw/`에 먼저 남기고, 주요 결정은 source 요약과 관련 feature/decision 페이지에 추적 가능하게 연결한다.
- 기능 명세가 바뀌면 "명세가 바뀌었다"는 사실도 raw나 회의록에 남겨야 한다.
- 디스코드 스레드는 별도 raw 후보로 수집하거나, 회의록에 요약해 넣은 뒤 ingest해야 한다.

## 미해결

- 디스코드 스레드를 누가, 어떤 단위로 raw화할지.
- PM Agent 또는 item-log 방식의 구체 운영 방식.
- 기능 명세 업데이트 알림을 어떤 채널/포맷으로 남길지.
