# 풀이관리

maker가 진행 중인 풀이 세션을 제어하는 화면. **선착순 모드(실시간)** + **학습 모드** 두 갈래.

문제셋 라이프사이클은 [features/problem-set-lifecycle.md](./problem-set-lifecycle.md). player 뷰는 [features/play-player.md](./play-player.md).

## 공통

- **답안 수정 정책**: [decisions/answer-edit-during-play.md](../decisions/answer-edit-during-play.md)
- **player 정보 표시**: 실명+닉네임 9자 제한(말줄임). 정답/오답 색상 구분(예: 빨강/파랑). 무한 스크롤.
- **player 탭**: (1) 선착순(제출 순) / (2) 정답자(선착순 기반) / (3) 오답자(선착순 기반).

## 선착순 모드 (실시간)

### 1.0 시작하기 / 종료하기

| 상태 | 버튼 | 효과 |
|---|---|---|
| 진행 예정 | "시작하기" | 클릭 시 → 진행 중 |
| 진행 중 | "종료하기" | 클릭 시 → 종료됨 + 복습 전환 흐름 |

시작하기 누르기 전에는 모든 문제에 시작하기가 노출.

### 1.3 문제 제어 토글

| 토글 | 효과 |
|---|---|
| **문제공개** | OFF면 player가 문제 내용을 볼 수 없음 |
| **제출허용** | ON 시 랜덤 시간 후 종 애니메이션 + 제출 버튼 활성화 / OFF면 제출 비활성화 |

**제약**: 문제공개 ON이어야 제출허용 ON 가능.

### 1.5 득점자

- 가장 빠르게 정답 제출한 사람.
- 인정답안 추가 등으로 정답이 바뀌면 **득점자 자동 갱신**.

### 1.6 진출자 선정

`진출자` 버튼 클릭 → 별도 페이지.

| 영역 | 동작 |
|---|---|
| 1) 선착순 기준 등수 | 빠르게 정답 많이 낸 순. 공동 등수는 가나다순 |
| 2) 정답자 기준 등수 | 선착순 무관, 정답 개수만. 동일 등수는 쉼표로 묶어 표시 |
| 3) 다음단계 진출자 영역 | 1)/2)에서 선택한 player 표시. **직접 입력 가능** (`이하은(웃긴 감자)` 형식 정확히) |

선택 방법: 체크박스 직접 선택 / "선착순 N등까지" 입력 시 1~N등 자동 체크.

확정 시 모달: "0000 진출자로 확정하시겠습니까?" → 확인 → 풀이관리 페이지로 복귀 / 취소 → 머무름.

### 1.7 문제 이동

- 상단 번호 클릭으로 직접 이동
- "이전문제" / "다음문제" 버튼

### 1.8 종료하기 (상세)

호버 시 "우승자 정보 player에게 공개하기" 안내. 클릭 → 팝업: "우승자는 000입니다. PLAYER 에게 결과를 공개하시겠습니까?"

| 버튼 | 흐름 |
|---|---|
| **전송하기** | player에게 우승자 알림 + maker도 동일 화면 → 종료하기 → 공개범위 설정(기존 설정 default) → 확인 → 복습 화면 |
| **종료하기** | 우승자 전송 없이 종료 → 공개범위 설정(기존 설정 default) → 확인 → 복습 화면 |

종료 후 player 측 우승자 화면에는 "풀이결과 보러가기" 진입점이 있어 [풀이결과 대시보드](./result-dashboard.md) 상세로 이동 가능.

## 학습 모드

### 풀이 시작
- "시작하기" 버튼 클릭으로 문제 활성화.

### 자동 종료
- **모든 인원이 풀기 완료 시 자동 종료.**
- 풀이 중 새 player가 가입한 경우, **종료 처리 이전에 들어온 player까지 팀원으로 간주** → 그 인원까지 다 풀어야 자동 종료.
- **수동 종료**도 가능.

### 1.4 문제 이동
- 상단 번호 클릭 / "이전문제" / "다음문제".
- 마지막 문제에서 "다음문제" → "**종료하기**"로 변경.
- 종료하기 → 공개범위 설정(전체/그룹/비공개) → 확인 → 복습 화면.

## 주간 회의에서 나온 후속 메모

- 2026-03-29 회의에서는 실시간 중간 참여자를 `풀 수 없는 상태`로 분명히 안내하고, 닉네임 기준 식별과 알림 토픽 연동이 필요하다고 정리했다. [2026-03-29 주간 회의록](../sources/2026-03-29-weekly-meeting.md)
- 2026-03-22 회의에서는 우승자 선정 팝업과 종료하기 흐름이 실제 운영에서 안정적으로 동작하는지 다시 검증해야 한다는 회고가 남았다. [2026-03-22 주간 회의록](../sources/2026-03-22-weekly-meeting.md)
- 같은 회의에서 사이드바 자동 접힘 기준은 단순 반응형 이슈가 아니라, 어떤 상황에서 조작을 숨길지에 대한 규칙 정의 문제로 다뤄졌다. [2026-03-22 주간 회의록](../sources/2026-03-22-weekly-meeting.md)
- 2026-02-22 회의에서는 우승자 선정 후 종료하기 팝업 미노출 문제가 배포 준비 항목으로 별도 점검됐다. [2026-02-22 주간 회의록](../sources/2026-02-22-weekly-meeting.md)
