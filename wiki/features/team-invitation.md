# 팀 사용자 초대

팀에 새 멤버를 추가하는 두 가지 방법: **직접 계정 추가** + **초대 링크**.

## 2.1 직접 계정 추가

| 단계 | 동작 |
|---|---|
| 진입 | 상단 `계정 추가` 버튼 |
| 검색 | 이메일 입력 — MAIT 가입자 내에서 검색. **정확한 이메일만 매칭** |
| 결과 | 닉네임 / 이름 / 이메일 표시 |
| 권한 선택 | 우측 드롭다운 (maker / player). 클릭 시 역할 설명 |
| 초대 | 이메일 + 권한 선택 후 `초대` 버튼 |
| 결과 | 즉시 해당 권한 리스트에 반영 |

### 알림
- 초대된 사용자에게 알림: "~~팀에 maker로 초대되었습니다."
- 알림 클릭 → 해당 팀의 문제 풀기 페이지로 이동.

## 2.2 초대 링크 생성

`초대 링크 생성` 버튼 → 모달:

| 설정 | 값 |
|---|---|
| 권한 유형 | maker / player |
| 유효기간 | 1일 / 3일 / 7일 |
| 승인 절차 | O / X — **player 선택 시 무조건 X 고정** |

## 2.3 초대 링크 결과

- `생성하기` → 링크 발급.
- 링크 텍스트 영역 + `복사하기` 버튼.
- 생성된 링크는 수정 불가 표시.

## 2.4 초대 링크 관리

하단 `초대링크 관리` 영역:

| 컬럼 | 내용 |
|---|---|
| 권한 | maker / player |
| 링크 | 복사 아이콘 |
| 종료 일시 | 만료 시각 |
| 삭제 | 삭제 아이콘 → 확인 모달 → 즉시 삭제 |

## 2.5 승인 절차 동작

| 초대 권한 | 승인 절차 가능 |
|---|---|
| maker | O (선택) — O 시 관리자 수락 후 등록 |
| player | **X 고정** — 자동 승인 |

## 2.6 초대 링크 접속 화면 (조건별)

| 링크 상태 | 로그인 | 승인 | 표시 버튼 |
|---|---|---|---|
| 만료 / 비유효 | — | — | 로그인하기 / 회원가입하기 |
| 유효 + 미로그인 | X | — | 로그인하고 참여하기 / 회원가입 후 참여하기 |
| 유효 + 로그인 | O | 불필요 | "참여하기" → 문제 풀기로 이동 |
| 유효 + 로그인 | O | 필요 | "확인" → 메인/이전 페이지. 승인 완료 시 헤더 알림 |
| 거절됨 | — | — | "홈으로 이동" |

승인 대기 사용자는 4번 페이지 안내(원본 명세에 4번 항목 명시 안 됨)를 모달로 안내.
