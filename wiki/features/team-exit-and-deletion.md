# 팀 탈퇴 / 팀 삭제

팀 멤버가 팀을 나가는 흐름과 팀 자체를 삭제하는 흐름. 멤버 권한 변경 UI는 [팀 멤버 권한 관리](./team-member-management.md), 초대 흐름은 [팀 사용자 초대](./team-invitation.md) 참고.

## 팀 탈퇴

- OWNER는 팀 탈퇴가 불가능하다.
- 일반 멤버가 탈퇴하면 해당 유저는 팀 페이지에 들어갈 수 없고, 팀 목록에도 노출되지 않는다.
- 탈퇴한 유저의 기존 데이터는 삭제하지 않고 유지한다.
  - 팀 유저 관계는 삭제 처리.
  - 해당 유저가 팀에서 풀었던 문제 기록과 생성했던 문제셋은 삭제하지 않는 방향으로 기록돼 있다.
- 탈퇴 완료 메일은 본인과 OWNER 중심으로 논의됐지만, MAKER까지 포함할지 여부와 정확한 문구는 확인이 필요하다.

근거: [2026-04-19 주간 회의록](../sources/2026-04-19-weekly-meeting.md).

## 팀 삭제

- 2026-04-19 회의에서는 논리 삭제를 고려하다가, 팀이 삭제됐는데 팀이 만든 문제셋이 남아있는 것이 더 어색하다는 이유로 **완전 삭제** 방향으로 정리했다.
- 삭제 범위 후보:
  - 팀
  - 팀 멤버 목록
  - 생성했던 초대링크 리스트
  - 가입 신청 내역
  - 실시간 제출 기록, 득점자, 학습모드 풀이 세션 및 임시 저장 데이터
  - 문제셋 및 문제
- 실시간 진행 중인 문제셋이 있어도 삭제하는 쪽으로 기록돼 있다.
- 2026-04-26 배포 후속에서는 팀 삭제 관련 미완성 데이터 보정 작업과 팀원 메일 수신 확인이 남았다.

근거: [2026-04-19 주간 회의록](../sources/2026-04-19-weekly-meeting.md), [2026-04-26 주간 회의록](../sources/2026-04-26-weekly-meeting.md).

## 배포 후 확인 항목

- 팀 탈퇴: OWNER와 본인에게 메일이 오는지 확인.
- 팀 삭제: 팀원에게 메일이 오는지 확인.
- 팀 삭제: 완전 삭제 범위에 포함된 데이터가 실제 화면/검색/API에서 남아 보이지 않는지 보정.

## 주간 회의에서 나온 후속 메모

- 2026-03-29 회의에서는 팀 탈퇴/삭제가 남은 정책 이슈로 언급됐다. [2026-03-29 주간 회의록](../sources/2026-03-29-weekly-meeting.md)
- 2026-04-19 회의에서 팀 탈퇴와 팀 삭제의 데이터 처리 방향이 구체화됐다. [2026-04-19 주간 회의록](../sources/2026-04-19-weekly-meeting.md)
- 2026-04-26 회의에서는 배포 후 팀 탈퇴/삭제 메일과 삭제 데이터 보정이 확인 항목으로 남았다. [2026-04-26 주간 회의록](../sources/2026-04-26-weekly-meeting.md)
