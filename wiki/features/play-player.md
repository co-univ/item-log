# 문제 풀기 (player 측)

player가 보는 풀이 화면. 실시간 / 학습 / 복습 3가지 모드.

maker 측 대응 화면은 [풀이관리](./play-management.md). 정답 표시 정책은 [decisions/scoring-display.md](../decisions/scoring-display.md).

## 1. 실시간 모드

### 1.1 문제 풀이 뷰

| maker 토글 | player 영향 |
|---|---|
| 문제공개 ON | 풀이 화면 접근 가능 |
| 제출허용 OFF | 제출 버튼 비활성화 |
| 제출허용 ON | **종 애니메이션** + 제출 버튼 활성화 |

선지 선택 또는 답안 입력 가능.

### 1.2 대기화면
maker가 문제공개 OFF인 상태. 풀이 불가, 대기.

### 1.3 다음 진출자
maker가 [진출자 선정](./winner-selection.md)을 한 경우의 화면.

### 1.4 채점

- 정답 / 오답 여부만 노출.
- **재풀이 정책**:
  - 정답 — 다른 답안 선택/입력 불가 (잠김).
  - 오답 — 재제출 가능. **단 풀이관리에서 정답자/오답자 동시 표기되는 이슈 미결.**

### 1.5 이탈 후 복귀

- 풀이 중 이탈 후 다시 들어와도 데이터 보존.
- 진행 중인(문제공개 ON된) 문제로 자동 이동.
- 풀지 않은 문제는 데이터 없음 (당연).
- (12/20) 이탈 사이에 종료된 경우에도 데이터 유지.

## 2. 학습 모드

### 문제셋별 상태 (4/11 추가)

| 상태 | 진입 조건 |
|---|---|
| 풀이 전 | maker가 시작했으나 아직 풀지 않음 |
| 풀이 중 | 풀고 있고 자동저장되는 중 |
| 풀이 완료 | 채점하기 버튼 누른 후 |

### 2.1 문제 번호

- 자유 이동.
- 푼 문제 / 안 푼 문제 시각 구분.
- **"풀었다" 기준**:
  - 객관식 — 1개 이상 선택
  - 주관식·빈칸 — 답안 영역 모두 채움 (다답일 때 1개만 채우면 풀지 않은 것)
  - 순서 — 모두 채움

### 2.2 풀이 뷰

- 제출 제한 없이 답 입력/선택.
- 제출하기 버튼은 모든 문제에 노출 — **문제 단위가 아니라 문제셋 전체 제출**.
- 제출 → 모달 → 확인 → 채점 뷰로.

### 2.3 채점 뷰

- 풀이 완료 상태로 전환, "통계확인" 버튼으로 변경.
- **정/오답 + 풀이 모두 한꺼번에 노출** (해설 즉시 노출, 복습과 다름).
- **재풀이 불가**.

### 2.4 자동저장

- 다른 문제로 이동 시 이전 문제 저장.

### 2.5 이탈 처리

- LNB·로고 클릭 등 페이지 이탈 시 팝업.
  - 취소 → 머무름.
  - 확인 → 저장된 부분만 보존, 페이지 이동.

## 3. 복습 모드

### 3.1 문제 번호
자유 이동.

### 3.2 풀이 뷰 — To-be 정책

| 액션 | 동작 |
|---|---|
| 제출하기 | 답안 없으면 제출 불가 |
| 제출 후 | 채점만 표시, **다시풀기 / 해설보기** 활성화 |
| 다시풀기 | 다시 풀이 가능 |
| 해설보기 | 해설 표시 |
| 해설닫기 | 해설 숨김, 채점만 |

(As-is 동작은 source 페이지의 raw 표 참고.)

### 3.3 이탈 후 복귀

- **24시간 내** 재진입 → 마지막 푼 문제 + 정답 OFF + 해설 OFF
- **24시간 초과** → 1번 문제 + 정답 OFF + 해설 OFF
- 12/10 결정 보강: 문제 번호 한 번 이동하거나 셋 이탈 후 재진입 시 정답·풀이 무조건 OFF로 리셋.

### 자유 노트
- **여러 번 풀이 가능** — 한 번만 풀고 끝나면 복습 의미가 없음.
- 복습 셋은 이미 종료된 것이므로 **문제 수정·풀이관리 버튼 불필요**.
