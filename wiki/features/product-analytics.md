# 제품 분석 / GA4 이벤트

GA4와 뷰저블 등 제품 사용 데이터를 통해 기능별 이탈과 클릭을 관측하는 영역. 2026-04-26 회의에서 다음 주 회의까지 준비할 항목으로 올라왔다.

## 2026-04-26 기준 지표 후보

- **로딩 UI 이탈률**: 사용자가 로딩 화면에 들어왔다가 나가는지 추적.
- **AI 문제 생성 시간**: API 처리 시간이 아니라 사용자가 화면에서 체감하는 생성 시간을 기준으로 측정.
- **문제셋 제목 수정 클릭**: 문제 관리 → 문제 카드 → 연필 아이콘 → 문제셋 제목 수정 경로에서 몇 명이 클릭하는지 확인.

근거: [2026-04-26 주간 회의록](../sources/2026-04-26-weekly-meeting.md).

## 2026-05-04 후속

- GA4에 추가할 기능 정리는 아직 할 일로 남았다.
- 이벤트를 카테고리화해서 컴포넌트에 적용 가능한지 여부를 구분해 보기로 했다.

근거: [2026-05-04 주간 회의록](../sources/2026-05-04-weekly-meeting.md).

## 2026-05-10 후속

- GA4 추가 기능 정리는 기존 `MAIT-124`로 재사용하기로 했다.
- 디자인 파트에서는 뷰저블 데이터가 아직 남은 작업으로 기록됐다.

근거: [2026-05-10 주간 회의록](../sources/2026-05-10-weekly-meeting.md).

## 2026-06-14 어드민 페이지 방향

- 어드민 페이지는 당장 사용자에게 보이는 기능보다 내부 지원 도구에 가깝다.
- GA나 뷰저블로 잡기 어려운 데이터, 유저 관리, FE QA 전용 API를 제공하는 방향이 확인됐다.
- 백엔드 후속으로 기획자에게 필요한 데이터 전달과 QA용 API 실행이 남았다.
- 기존 `MAIT-64`에서 계속 추적한다.

근거: [2026-06-14 주간 회의록](../sources/2026-06-14-weekly-meeting.md).

## 연결 기능

- [AI 문제 생성](./ai-question-generation.md): 생성 대기 시간이 길고 실패 가능성이 있어 체감 시간과 이탈률 관측이 필요하다.
- [문제셋 생성 플로우](./problem-set-creation.md): AI 생성 진입과 편집 진입이 연결된다.
- [문제셋 라이프사이클](./problem-set-lifecycle.md): 문제 관리 카드와 제목 수정 흐름이 포함된다.
- 어드민 페이지: GA/뷰저블 외 내부 운영 데이터와 QA 지원 API가 연결된다.

## 미해결

- GA4 이벤트명, 파라미터, 수집 시점.
- 뷰저블과 GA4를 각각 어디까지 사용할지.
- 뷰저블 데이터는 어떤 화면/행동을 기준으로 볼지.
- 화면 기준 시간 측정의 시작/종료 지점.
- 이벤트를 어떤 카테고리 체계로 묶고, 어떤 컴포넌트에 적용 가능한지.
- 어드민에서 제공할 기획자용 데이터와 FE QA용 API 범위.
