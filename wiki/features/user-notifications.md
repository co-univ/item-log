# 알림

MAIT에서 사용자에게 팀/풀이 상태 변화를 알려주는 기능. 2026-04-26 회의에서는 `MAIT-126`으로 필요한 알림 지점을 정리하기로 했고, 2026-05-04 회의에서 역할별 수신자와 클릭 후 이동 규칙이 구체화됐다.

2026-05-26 회의에서는 6월 2일 배포에서 일정상 일부 알림 항목을 제외하고 6월 16일 추가 배포로 넘기기로 했다. 알림은 백엔드 우선 진행, 프론트 반영은 추후 진행으로 정리됐다. 근거: [2026-05-26 주간 회의록](../sources/2026-05-26-weekly-meeting.md).

## 팀 초대 / 팀 변경 알림

| 대상 | 상황 | 클릭 후 이동 |
|---|---|---|
| owner | 팀 초대 완료: `000님 초대가 완료` | 멤버관리 페이지 |
| owner | 팀 탈퇴: `000님이 팀을 탈퇴` | 멤버관리 페이지 |
| maker, player | owner가 팀 삭제 | 이동 없음 |
| maker, player | 본인이 팀 탈퇴 | 이동 없음 |

근거: [2026-05-04 주간 회의록](../sources/2026-05-04-weekly-meeting.md).

기존 팀 초대 명세에는 초대된 사용자에게 `~~팀에 maker로 초대되었습니다.` 알림을 보내고, 클릭 시 해당 팀의 문제 풀기 페이지로 이동한다고 적혀 있다. 2026-05-04 회의의 owner 대상 `초대 완료` 알림과는 대상과 이동 위치가 다르므로 함께 유지한다. 근거: [2026-04-14 팀관리 명세](../sources/2026-04-14-spec-team-member-and-category.md).

## 풀이 / 채점 알림

- 인정답안 추가로 인해 기존 오답이 정답으로 바뀐 player에게 알림을 보낸다.
- 학습모드에서는 제출 이후 알림이 뜨고, 클릭 시 해당 문제의 채점된 화면으로 이동한다.
- 실시간모드 케이스는 항목만 열려 있고 상세 이동/노출 규칙은 아직 없다.

근거: [2026-05-04 주간 회의록](../sources/2026-05-04-weekly-meeting.md).

## 미해결

- 실시간모드에서 인정답안 변경으로 오답이 정답이 된 경우 알림 문구와 클릭 후 이동 위치.
- 팀 삭제/탈퇴 알림이 이동 없음일 때 사용자가 어디에서 내역을 다시 확인할 수 있는지.
- 기존 초대된 사용자 알림과 owner 대상 초대 완료 알림의 발송 시점과 중복 여부.
- 6월 2일 배포에서 제외돼 6월 16일 추가 배포로 이동한 알림 항목의 정확한 범위.
