# 문제 셋 라이프사이클

문제 셋은 다음 상태를 가진다. 메인 화면(문제 관리 페이지)은 이 상태별로 섹션이 나뉘어 노출된다.

## 상태 다이어그램

```
[제작 중]                                          ← 임시저장만 한 상태
   │
   ├─ 생성하기(실시간 풀이) ──→ [진행 예정] ──시작하기──→ [진행 중] ──종료하기──→ [종료됨]
   │                                                                              │
   │                                                                       ┌──────┴──────┐
   │                                                                    재시작        복습 이동
   │                                                                       │             │
   │                                                                  [진행 중]      [복습]
   │
   └─ 생성하기(학습 풀이) ──────────────────────────────────────────────→ [학습풀이]
```

## 각 상태별 동작

### 1. 제작 중 (임시저장)
- 임시저장만 하고 생성하기를 안 한 상태
- 미명명 시 `문제셋1, 문제셋2, ...` 자동 명명
- 자동저장본만 있고 생성하기를 안 했으면 `임시 저장본n`으로 노출 (raw §3.11)

### 2. 실시간 풀이 — 진행 예정
- 생성하기 완료, 아직 시작하기 누르지 않음

### 2. 실시간 풀이 — 진행 중
- 시작하기 누른 상태, player가 실시간으로 풀이 참여 중

### 2. 실시간 풀이 — 종료됨
- 종료하기 버튼 누른 상태 — [풀이관리 1.8](./play-management.md) 참고.
- **maker는 접근 불가** (편집 불가)
- 종료 시 흐름:
  - **우승자 공개 옵션** ("PLAYER에게 결과를 공개하시겠습니까?") → 전송하기 / 종료하기.
  - **공개범위 설정 모달** (기존 셋 설정으로 default 체크).
  - 확인 → 복습 화면.
- 두 가지 후속 액션:
  - **재시작** — 문제 풀이 다시 시작. *데이터 처리 정책 미확정* (raw §1.2)
  - **복습상태로 이동** — 공개 범위 선택 후 복습으로 전환

### 3. 학습 풀이
- 생성 시 학습 풀이를 선택하고 생성하기 완료된 상태.
- **자동 종료**: 모든 인원이 풀기 완료 시 자동 처리. 풀이 중 합류한 player까지 포함해서 다 풀어야 종료.
- **수동 종료**도 가능. 마지막 문제에서 "다음문제" → "종료하기"로 변경.
- 종료 시: 공개범위 설정(전체/그룹/비공개) → 확인 → 복습 화면.

### 4. 복습
- 실시간/학습 풀이가 끝난 문제 셋
- 공개 범위 변경 가능 — [visibility-states](../decisions/visibility-states.md) 참조
- 검색 가능한 유일한 상태 — [problem-search](./problem-search.md) 참조
- 복습 셋은 문제 수정·풀이관리 버튼 불필요 (자유 노트, 12/10)

## 모든 섹션 공통 동작

- 문제셋 목록에 **제목, 최종 수정일, 문제 수정** 노출
- 제목 인라인 편집: 연필 클릭 → 텍스트 편집 → Enter 저장
- 삭제: 휴지통 클릭 → '문제 셋을 삭제하시겠습니까?' 모달 → 확인/취소
- 문제 수정 클릭 → 질문 편집기로 이동

## 주간 회의에서 나온 운영 메모

- 2026-03-29 회의에서는 문제셋 삭제를 `풀이 전 / 진행 중 / 완료 후` 상태마다 다르게 봐야 한다는 의견이 정리됐다. 실시간 모드 삭제는 대시보드 통계와 플레이어 경험까지 함께 고려해야 한다. [2026-03-29 주간 회의록](../sources/2026-03-29-weekly-meeting.md)
- 2026-03-29 회의에서는 학습모드를 실시간모드의 단순 변형이 아니라 개인 학습과 팀 스터디를 함께 담는 더 넓은 흐름으로 봐야 한다는 의견도 나왔다. [2026-03-29 주간 회의록](../sources/2026-03-29-weekly-meeting.md)
- 2026-01-11과 2026-01-03 회의에서는 비공개 문제셋 접근, 복습 해설, 완료된 문제셋 재확인성, 재채점처럼 상태 전환 이후 규칙이 사용자 경험에 직접 영향을 주는 이슈가 지속적으로 보고됐다. [2026-01-11 주간 회의록](../sources/2026-01-11-weekly-meeting.md), [2026-01-03 주간 회의록](../sources/2026-01-03-weekly-meeting.md)

## 미해결

- 재시작 시 이전 풀이 데이터 보존 여부 / 새 세션 분리 여부
- 복습 상태에서 전체공개일 때 비-player(검색으로 진입한 사용자)의 관리 흐름

- 학습모드가 개인 학습과 팀 스터디를 어떻게 함께 포괄하는지에 대한 상세 규칙
