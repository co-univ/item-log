# 질문 편집기

문제셋 생성 3단계의 핵심 화면. 4가지 질문 유형을 편집·미리보기·임시저장·생성하기까지 처리.

## 공통 동작

| 기능 | 동작 |
|---|---|
| 3.1 순서 변경 | 질문 번호 영역 드래그 앤 드롭 |
| 3.2 질문 삭제 | 호버 시 minus 아이콘 노출 |
| 3.3 질문 추가 | plus 아이콘 — 추가 시 객관식 4선지 default |
| 3.4 질문 입력 | (워딩: "문제" → "질문"으로 통일 예정) |
| 3.5 이미지 추가 | 한 질문당 이미지 1개 |
| 3.13 미리보기 | player 노출 뷰 그대로 |
| 3.14 PDF 다운 | 문제셋 PDF 출력 |

## 4가지 유형

### 3.6 객관식

- 디폴트 4선지, 최대 8선지.
- **선지 최소 2개** — 2개일 때 삭제 시도 차단.
- 체크박스 선택 → 정답 영역에 선지 번호 노출.
- 해설 입력 가능 (최대 자수 미정).

### 3.7 주관식

- 디폴트 답안 1개, 최대 5개.
- **답안 최소 1개** — 1개일 때 삭제 차단.
- 답안마다 **인정답안** 추가 가능 (최대 5개). 필드 이탈 시 저장.
- 해설 입력 가능 (최대 자수 미정).
- 정답 영역: 답안만 노출 (인정답안 제외).
- **정답 개수 조절 모드** (토글):
  - on: player에게 요구되는 답안 개수를 1~5로 조절. 기본값은 maker 답안 개수.
  - off: 조절 불가.
  - 설명 텍스트: "정답의 개수과 player에게 요구되는 답안의 개수가 다를 때 사용하는 모드입니다."

### 3.8 빈칸 넣기

- 질문 작성 도중 "빈칸 추가" 클릭 → 빈칸 삽입.
- 번호 규칙: 텍스트상 앞에 있는 빈칸이 앞 번호. 아래 답안도 빈칸 순서에 맞게 자동 재정렬.
- 빈칸 삭제: 백스페이스 1회 → 선택 표시, 2회 → 삭제 / minus 아이콘 클릭.
- 답안마다 인정답안 추가 가능 (최대 5개).
- 정답 영역 표시: `(1) A, (2) B` 형식.
- 디자인 노트: "답안추가" → "빈칸추가"로 워딩 개선.

### 3.9 순서

- 디폴트 답안 3개, 최대 6개.
- **보기 필드와 정답 필드 분리.**
  - 보기: 답안 생성에 맞춰 자동으로 알파벳 부여 (변경 불가).
  - 정답: 보기 내용을 직접 입력하지 않고 **알파벳을 입력**.
- 정답 영역 표시: `A, B, C, D` 형식.

## 저장 / 생성

| 액션 | 동작 |
|---|---|
| 3.10 임시저장 | 최초 임시저장 시 제목·주제 입력 강제. 이후는 단순 "저장". 변경 없으면 비활성화 |
| 3.11 자동저장 | 편집 시점부터 활성화. 변경 발생 후 1분 뒤 저장. 생성하기 미실행 + 화면 이탈 시 "임시 저장본n"으로 메인에 노출 |
| 3.12 생성하기 | 라이프사이클 확정. validation 처리 — [problem-set-creation.md](./problem-set-creation.md) 참고 |
| 3.15 AI 추천 더 받기 | 유형·개수 선택 → 추가 질문 생성. 개수 미입력 시 유형당 2개 default |

저장 정책 상세는 [decisions/auto-save-behavior.md](../decisions/auto-save-behavior.md) 참고.
