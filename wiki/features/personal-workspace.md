# 개인 워크스페이스

팀이 없거나 로그인 전 `바로 시작하기`로 들어온 사용자를 위한 개인 단위 공간. 팀 기반 권한/멤버 관리가 없고, 사용자는 maker에 해당하는 것으로 처리된다.

## 진입

- 팀이 없는 사용자가 로그인하면 개인 워크스페이스를 자동 생성하고 해당 워크스페이스로 이동한다.
- 로그인 전 `바로 시작하기` 클릭 → 로그인 → 개인 워크스페이스 생성 및 이동.
- 팀이 있는 경우 최근 방문 팀 기준으로 이동한다.
  - player: `문제 풀기` 탭으로 이동
  - maker: `문제 관리` 탭으로 이동
  - 개인 워크스페이스: maker에 해당하므로 항상 `문제 관리`로 이동
- 최근 방문 팀 저장 작업은 별도 후속으로 남아 있다.

근거: [개인 워크스페이스 명세서](../sources/2026-05-10-spec-personal-workspace.md).

## 문제 관리

- 본인이 모든 문제 풀이를 완료하면 완료 상태로 변경한다.
- 미트볼 및 버튼 UI는 기존 문제 관리와 동일하게 유지한다.
- **실시간 모드는 제외**한다.
- **풀이 시작 및 종료 기능은 제외**한다.
- 문제 풀이 완료에 따른 자동 종료 처리는 유지한다.
- Maker의 수동 종료 처리는 불가능해야 한다.

즉 개인 워크스페이스의 학습 문제셋은 팀 학습모드처럼 maker가 풀이를 시작/종료하며 팀원 상태를 관리하는 흐름이 아니다. 명세가 `풀이관리` 탭명 자체를 직접 언급하지는 않지만, 풀이 시작/종료 기능은 적용 대상에서 빠진다. 다만 본인이 모든 문제 풀이를 완료했을 때 완료/통계 전환으로 넘어가는 자동 처리는 살린다. [2026-05-31 주간 회의록](../sources/2026-05-31-weekly-meeting.md)

## 문제 풀기

- 학습 모드와 복습 모드만 제공한다.
- 실시간 풀이 기능은 제외한다.

## 풀이 결과 대시보드

- 랭킹 영역은 제외한다.
- 정답률 영역을 상단에 배치한다.
- 풀이 완료 후 통계로 전환한다.
- 상세 대시보드에서는 오답률을 노출하지 않는다.

## 카테고리 / 팀관리

- 개인 워크스페이스에서는 `팀관리`를 `카테고리 관리`로 명칭 수정한다.
- 카테고리 관리 화면은 기존과 동일하게 유지한다.
- 멤버관리는 비활성화한다.

## 주간 회의에서 나온 후속 메모

- 2026-04-26 회의에서는 개인 워크스페이스가 명세 추가, 디자인 검토, 백엔드 API 작업 시작 상태로 언급됐다. [2026-04-26 주간 회의록](../sources/2026-04-26-weekly-meeting.md)
- 2026-05-04 회의에서는 5월 12일 배포 후보로 개인 워크스페이스가 포함됐다. [2026-05-04 주간 회의록](../sources/2026-05-04-weekly-meeting.md)
- 2026-05-10 회의에서는 개인 워크스페이스 명세서 링크가 기획 회의 항목으로 공유됐다. [2026-05-10 주간 회의록](../sources/2026-05-10-weekly-meeting.md)
- 2026-05-14 회의에서는 다음 배포 범위를 기존 Jira 이슈 재사용 원칙으로 정리하면서 개인 워크스페이스를 기존 활성 이슈와 연결하기로 했다. [2026-05-14 주간 회의록](../sources/2026-05-14-weekly-meeting.md)
- 2026-05-26 회의에서는 회원 가입 시 자동 생성, 기존 유저 자동 생성, 실시간 모드 문제셋 생성 차단, 초대링크 생성 차단이 완료 체크됐다. 백엔드 할 일에서도 팀명 수정 API 확인, 마이그레이션 잡, 기본 생성 워크스페이스 닉네임 기준 변경이 완료 체크됐고, 기존 `MAIT-106`에 회의 후속을 연결했다. [2026-05-26 주간 회의록](../sources/2026-05-26-weekly-meeting.md)
- 2026-05-31 회의에서는 풀이 시작/종료 기능이 제외되더라도 문제 풀이 완료에 따른 자동 종료는 유지하고, Maker 수동 종료는 불가능해야 한다고 확인했다. 풀이 완료 후에는 통계로 전환하고, 개인 워크스페이스 상세 대시보드에서는 오답률을 노출하지 않는다. [2026-05-31 주간 회의록](../sources/2026-05-31-weekly-meeting.md)

## 미해결

- 최근 방문 팀 저장 기준과 개인 워크스페이스 포함 여부.
- 개인 워크스페이스에서 `풀이관리` 탭명을 UI에서 숨길지, 기존 문제 관리 UI 안에서 시작/종료 액션만 제거할지.
- 완료 상태 이후 복습 전환/공개 범위 설정을 팀 학습모드와 동일하게 볼지. 단, 풀이 완료 후 통계 전환은 2026-05-31 회의에서 확인됐다.
- 기본 생성 워크스페이스 명칭을 닉네임 기준으로 바꾸는 정책이 기존 팀명/워크스페이스명 수정 API와 충돌하지 않는지.
