# 로그인 / 회원가입 / 마이페이지

mait 사용자 진입과 계정 관리 영역. 정책 디테일은 [decisions/account-policies.md](../decisions/account-policies.md) 참고.

## 로그인

### 1.1 소셜 로그인 (구글/카카오/네이버)

- 인증 성공 → 사용자 이름·이메일 수신.
- 인증 실패 (취소·네트워크·권한거부) → 표준 에러 토스트 + 재시도 안내.

### 1.2 로컬 로그인 (이메일 + 비밀번호)

- 이메일·비밀번호 입력. **눈 아이콘**으로 비밀번호 보기/숨기기.
- "로그인 유지" 옵션.
- 입력값이 모두 있으면 로그인 버튼 활성화.
- 실패 안내: "아이디 또는 비밀번호가 올바르지 않습니다."

## 회원가입

### 2.1 소셜 로그인 최초 진입 → "계정 설정하기" 모달

- 약관 동의(필수/선택).
- 닉네임 입력 — 자동 랜덤 제안 + 중복확인 버튼.
- 필수 동의 + 닉네임 유효 → "시작하기" 활성화 → 메인 우측 상단 "로그인 → 이름"으로 변경.
- "닫기" 클릭 시 로그인 실패 처리.

### 2.2 로컬 회원가입

순서대로 입력 — 모든 필드 통과해야 가입 가능.

| 필드 | 핵심 동작 |
|---|---|
| 이메일 | 형식 검증(`@`/도메인/공백) → 중복 체크 → 인증메일 발송(10분 유효) → 메일 내 "인증하기" |
| 비밀번호 | 8자 이상 + 영문 + 숫자 + 특수문자. 조건 체크박스 실시간 표시(회색→초록), 눈 아이콘 |
| 비밀번호 확인 | 실시간 일치 검증 |
| 이름 | 한글/영문만, 2~20자. 특수문자·숫자 불가 |
| 전화번호 | 숫자만, 중복 불가 |
| 닉네임 | 자동 생성(`user1234`), 2~20자, 금지어 필터링, 중복 확인 |
| 약관 | 서비스 이용약관 + 개인정보 수집 (모두 필수) |

가입하기 버튼: 위 모든 검증 + 인증 + 약관 통과 시 활성화. 중복 클릭 방지(로딩 표시) → 성공 시 자동 로그인 + 초기 화면 이동.

## 마이페이지

### 3.1 개인 정보 확인

- 표시 항목: 이메일, 이름, 전화번호, 닉네임, 비밀번호.
- **이메일·이름은 수정 불가** (가입 시 입력값 그대로).

### 3.2 개인 정보 수정

| 항목 | 동작 |
|---|---|
| 전화번호 | 숫자만, 중복체크 버튼, 중복 시 오류 |
| 닉네임 | 2~20자, 금지어 필터, 중복체크 또는 실시간 검증 |
| 비밀번호 | 기존 → 신규 → 확인. 영문+숫자+특수문자 8자+ 조건 + 일치 검증 |

수정 완료 시 성공/실패 메시지 + 변경 반영.
